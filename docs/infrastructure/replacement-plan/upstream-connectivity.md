### ISP回線の断片化

推定で最低3回線（NTT系 × 2 + オプテージ × 1）が存在。北野校舎だけでもルーター2台が確認されており、棟ごとまたはフロアごとに個別の回線契約がある可能性が高い。これにより：

- **帯域の非効率な分散**: 各回線が個別契約のため、トラフィックの偏りがあっても融通できない
- **管理コストの増大**: 契約・障害対応・請求が回線ごとに分散
- **ルーティングの複雑化**: フロアごとに異なるデフォルトゲートウェイが存在し、統一的なネットワーク設計が不可能
- **セキュリティポリシーの不統一**: 各回線で異なるNAT/FW設定が運用されている可能性

### ボトムアップ試算

#### シナリオ1: 通常授業時

```
ピーク在校 1,200人 × 端末係数 2.0 = 2,400台

内訳（推定）:
  ヘビー（ゲーム・CG・サウンド・CAD・IT）: 700人 × 平均 15 Mbps = 10.5 Gbps
  ライト（ビジネス・国際・日本語 + BYOD）:  500人 × 平均  5 Mbps =  2.5 Gbps
  BYOD端末（バックグラウンド通信）:       1,200台 × 平均  2 Mbps =  2.4 Gbps
                                                          合計 ≒ 15.4 Gbps
  コンテンション比 3:1 適用 → 実効必要帯域 ≒ 5 Gbps
```

**注:** 「平均15 Mbps」はヘビーユーザーの**持続的な平均**であり、バースト（ゲームエンジンDL中など）は100 Mbps以上に達する。コンテンション比3:1は「全員が同時にピークを出すわけではない」補正。

#### シナリオ2: 一斉更新イベント（最悪ケース）

```
授業開始時にUnreal Engine/Unity/Adobe CCの一斉アップデートが走る場合:
  教室40台 × 100 Mbps = 4 Gbps（1教室で）
  同時に3〜5教室で発生 → 12〜20 Gbps

Windows Update配信日（Patch Tuesday）:
  1,000台 × 数GB = 推定ピーク 5〜10 Gbps
```

一斉更新はWSUS/SCCM等のキャッシュサーバで緩和可能だが、現状そのような基盤がないためインターネット直接DLになる。

#### シナリオ3: ハイブリッド授業・配信

学校が推進している「ハイブリッド型授業・オンデマンド配信」が本格化した場合:
```
  配信教室 10室 × 上り 10 Mbps = 100 Mbps（上り帯域が重要）
  受講側は外部からのアクセスのため上流帯域に影響
```

これ単体では大きくないが、他のトラフィックに上乗せされる。

### 他校事例: 東京国際工科専門職大学（IPUT）

出典: 藤井竜也「専門職大学の教育に向けたITインフラの設計と構築」東京国際工科専門職大学紀要 第1巻第1号
https://iput-t.repo.nii.ac.jp/record/34/files/10_%E7%B7%8F%E8%AA%AC_%E8%97%A4%E4%BA%95%E7%AB%9C%E4%B9%9F.pdf

IPUTは神戸電子と**学科構成がほぼ同じ**（ゲーム/CG/AI/情報工学）の専門職大学であり、帯域設計の参考になる。

| 項目 | IPUT | 神戸電子 | 比較 |
|------|------|---------|------|
| 学生定員 | 800名（完成年度） | 1,970名 | **神戸電子が2.5倍** |
| 学科構成 | デジタルエンタテインメント（ゲーム/CG）、情報工学（AI戦略） | ゲーム・グラフィックス・サウンド・CAD・IT・ハード・ビジネス等 | ほぼ同種、神戸電子のほうが多様 |
| 校舎 | モード学園コクーンタワー（1棟、他校と同居） | 12棟（分散） | IPUTは1棟で済む分シンプル |
| 基幹LAN | **10GbE済** | 100BASE-TX（RT107e）〜1GbE | **100倍以上の差** |
| PC教室 | 約300台（64GB RAM, RTX 2070/Quadro RTX4000） | 不明（調査必要） | — |
| 学内サーバ | ファイルサーバ（NetApp 100TB）、AIサーバ（V100×20基）、レンダリングサーバ、HCI（Nutanix） | 「Linuxサーバ群」（詳細不明） | — |

**IPUTの帯域に関する知見（セクション5.3「IPネットワークの容量問題」）:**

> 本校舎では、各学校のオンライン化によってIP通信トラフィックの**輻輳が常態化**している。（中略）学内のサーバ活用のために校舎内の基幹LANは**10GbE化が完了**しているが、YouTube などを教材として利用する授業など、インターネット接続が前提の授業が増えるため**この増速は緊急の課題**となっている。

つまり**800名規模・10GbE基幹LAN・大容量サーバをオンプレミスに持つIPUTですら、対外接続の帯域不足が「緊急の課題」**と報告されている。

**IPUTの設計判断: オンプレミス vs クラウドの基準**

> サーバをオンプレミスに置くかクラウドに置くかの判断基準は、**IPネットワークに過大な帯域を消費するか否か**

大容量データ（映像/CG素材/AI学習データ）はクラウドに置くとインターネット接続点がボトルネックになるため、**意図的にオンプレミスのファイルサーバ（100TB）に置いている**。これはインターネット帯域が有限であることを前提とした設計。

**神戸電子への示唆:**

1. IPUTの2.5倍の学生数で、学科構成も同等以上に重い → IPUTが10GbEで足りていないなら、神戸電子は**10Gbpsが絶対的な最低ライン**
2. IPUTは1棟に集約されているため基幹LANの10GbE化が容易だったが、神戸電子は12棟分散 → 棟間バックボーンも10GbE以上が必要
3. IPUTは大容量データをオンプレミスに逃がすことでインターネット帯域を節約しているが、神戸電子にはそのようなサーバインフラすら存在しない → **全トラフィックがインターネット経由**になるため、帯域需要はIPUT以上
4. IPUTでは有線LAN接続を基本としている（バーストトラフィックが多いため）→ 神戸電子がWiFi前提で設計する場合、AP→スイッチ→上流の全経路で帯域を確保する必要がある

### KICとの比較

| | KIC（大学院） | 神戸電子（専門学校） | 倍率 |
|---|---|---|---|
| 学生数 | 少（大学院規模） | 1,755名 | 10倍以上 |
| トラフィック特性 | 研究・論文中心 | コンテンツ制作（ゲーム/CG/サウンド） | 重い |
| 回線 | SINET6 1Gbps | ? | — |

KICの1Gbpsは大学院規模には十分だが、神戸電子は人数が10倍以上、かつトラフィック特性がはるかに重い。**単純な人数比だけでも10Gbpsが最低ライン**。

### 帯域の推奨値

| 帯域 | 評価 |
|------|------|
| **1 Gbps** | 完全に不足。現状の抑圧された需要すら捌けない。KIC（1/10の規模）と同等では話にならない |
| **10 Gbps** | **最低ライン**。IPUTが800名で10GbEでも帯域不足を報告しており、2.5倍の学生数では下限。通常授業は回るが一斉更新や成長に余裕なし |
| **10 Gbps × 2本** | 推奨。冗長性 + 帯域分散。ISPを別事業者にすれば障害時のフェイルオーバーも確保 |
| **25〜40 Gbps** | 5年後を見据えるなら理想的。SINET6は10G/40G/100Gアクセスを提供しており、SINET接続が実現すればこのレンジも現実的 |

### 補足: 抑圧された需要

現在のインフラ（100BASE-TXのRT107e、頻繁な障害）では**ネットワークを使わない前提で授業が設計されている**可能性が高い。

なお、本館にはオンプレミスサーバが存在する（公開情報の「Linuxサーバ群によるインターネット接続校内LAN環境」の実体と推定）。ただしその規模・用途・帯域への影響は不明であり、IPUTのように意図的にトラフィックをオフロードする設計がされているかは確認が必要。インフラ改善後は追加のトラフィックがインターネット経由となるため：

- ゲーム系がクラウドビルドやアセットストリーミングを導入
- CG系がクラウドレンダリングを活用
- 全学科でAI関連ツール（LLM API、GPU クラウド）の利用が増加
- ハイブリッド授業・オンデマンド配信が本格化

つまり**リプレース直後から需要が急増する**ことを織り込む必要がある。現状のトラフィックを基準にサイジングすると確実に不足する。

### 結論

**10Gbps以上がmust**。推奨は10G × 2本（冗長 + 帯域）。SINETが利用可能になれば10G〜40Gの単一接続も選択肢。

## リプレース方針

### 目標構成

遠隔棟（グループ3）の接続方法により、アーキテクチャが2パターンに分かれる。

#### パターン1: 集約型（本館から遠隔棟へダークファイバ）— 推奨

本館に強い回線を1本引き、遠隔棟へはNTTダークファイバ等で本館から延伸する。全棟が1つのL3ドメインに収まる。

```
                    ┌─────────────┐
                    │  ISP / SINET │
                    └──────┬──────┘
                           │
                    ┌──────┴──────┐
                    │  コアルーター  │  本館MDF設置
                    │  (L3スイッチ)  │  FW/NAT/DHCP/DNS/VPN統合
                    └──────┬──────┘
                           │
         ┌─────────────────┼──────────────┐
         │                 │              │
    本館棟群(G1)      ┌────┴────┐    ┌───┴───┐
    ①②(館内配線)     │ ④北野館 │    │③東館  │
                     │ (G2)    │    │(G2)   │
                     └─────────┘    └───────┘
                     既存架空ファイバ   既存架空ファイバ
         │           (①↔⑤↔④,        (①↔③, ③↔④隣接)
    ┌────┴────┐       ①↔④)
    │ ⑤南館   │
    │ (G2)    │
    └─────────┘
    既存架空ファイバ
         │
    ┌────┼──────────────────┐
    │    │                  │
 ┌──┴──┐ ┌──┴──┐        ┌──┴──┐
 │共創館│ │一宮館│        │MM館 │  ...
 │(G3) │ │(G3) │        │(G3) │
 └─────┘ └─────┘        └─────┘
 NTTダークファイバ / 架空ファイバ / 無線ブリッジ（本館起点）
```

**メリット:** ルーティングがシンプル。全棟で統一的なポリシー・VLAN設計が可能。月額コストが回線1本分で済む。
**条件:** 本館→各遠隔棟のNTTダークファイバが利用可能であること（NTTに照会が必要）。

#### パターン2: 分散型（遠隔棟に個別回線 + OSPF/BGP統合）

各遠隔棟にそれぞれISP回線を引き、ルーティングプロトコルで仮想的に1つのネットワークに統合する。

```
    ┌─────────────┐        ┌─────────────┐       ┌─────────────┐
    │  ISP（主回線）│        │ ISP（回線2） │       │ ISP（回線3） │
    └──────┬──────┘        └──────┬──────┘       └──────┬──────┘
           │                      │                     │
    ┌──────┴──────┐        ┌──────┴──────┐       ┌──────┴──────┐
    │  本館コア    │◄─OSPF─►│  共創館ルータ │◄OSPF─►│  一宮館ルータ│
    │  ルーター    │  (VPN)  │             │ (VPN) │             │
    └──────┬──────┘        └──────┬──────┘       └──────┬──────┘
           │                      │                     │
      本館棟群+G2             共創館LAN              一宮館LAN
```

**メリット:** NTTダークファイバが利用できない場合でも実現可能。各棟が独立した対外接続を持つため冗長性が高い。
**デメリット:** 回線の月額固定費が棟数分かかる。各棟にルーターが必要。OSPF/BGP over VPN（IPsec）の設計・運用が複雑化。ネットワークポリシーの統一が難しくなる。

#### 判断基準

| 条件 | → 選択 |
|------|--------|
| NTTダークファイバが本館→遠隔棟で利用可能 | **パターン1（集約型）** |
| ダークファイバ不可だが自営管路 or 敷地内配線が可能 | **パターン1（集約型）** |
| 物理配線が一切不可能（公道・景観規制等） | **パターン2（分散型）** or 無線ブリッジ併用 |
| 遠隔棟の回線コストが十分安い | パターン2も選択肢に入る |

### Option A: SINET接続（推奨、要調査）

KIC（神戸情報大学院大学）は既にSINET6に1Gbpsで加入済み。同一法人（学校法人コンピュータ総合学園）として神戸電子専門学校をSINETに収容できる可能性がある。

**メリット:**
- eduroamとの親和性が最も高い（eduroam JPインフラがSINET上で運用されている）
- SINET6は10G/40G/100Gアクセスを提供しており、帯域要件（10Gbps以上）を満たせる
- 対外接続の帯域・品質が商用ISPより優位（学術ネットワーク、ベストエフォートではない）
- ISP契約の集約・コスト削減
- IPv6ネイティブ対応

**確認が必要な点:**
- 専修学校単独ではSINET加入資格がない → 法人名義または KIC加入の付帯としての接続可否
- SINET利用規程上、専門学校の一般トラフィック（学生のWeb利用等）が利用可能か
- KICの現在の1Gbps接続を法人全体で10Gbps以上にアップグレードできるか
- 物理接続: KICから神戸電子への専用線 or 独立したSINET接続が必要か

**調査ルート:** KIC教授 → SINET担当者（NII）

### Option B: 商用ISP集約

SINETが利用できない場合の代替。現在のフロア別バラバラ契約を10Gbps級の法人向け回線に集約する。

**候補構成:**
- 法人向け光回線 10Gbps × 1本（主回線）
- 別事業者の回線 × 1本（冗長用、別経路。1G〜10G）

**選定基準:**
| 項目 | 要件 |
|------|------|
| 帯域 | **10Gbps以上**（帯域試算より。ベストエフォートではなく帯域保証型が望ましい） |
| IPv6 | ネイティブデュアルスタック必須（eduroam + 将来のIPv6 only対応） |
| 固定IP | RADIUSサーバ公開用に必須（eduroam JP登録にIPアドレスが必要） |
| SLA | 障害時の復旧目標が明確であること |
| 冗長 | 主回線と別事業者・別経路の2本体制（予算次第） |

**見積もり目安（参考）:**
| 回線種別 | 帯域 | 月額目安 |
|---------|------|---------|
| フレッツ光クロス（法人） | 10Gbps（ベストエフォート） | 6〜7万円/月 |
| NURO Biz 10G | 10Gbps（ベストエフォート） | 18万円/月前後 |
| 法人向け帯域保証型（NTT Biz等） | 1〜10Gbps保証 | 30〜100万円/月 |
現在フロアごとにバラバラの契約を維持している総コストと比較すれば、10G 1本に集約しても大差ないか安くなる可能性が高い。現状の推定回線数（NTT × 2 + オプテージ × 1、おそらくさらに多い）の月額合計を確認した上で比較する。

### Option C: ハイブリッド（SINET + 商用ISP）

- 学術トラフィック（eduroam、研究用途）: SINET経由
- 一般トラフィック（学生のWeb閲覧、コンテンツDL等）: 商用ISP経由
- ポリシーベースルーティングで振り分け

SINETの利用規程上、一般トラフィックの通過が制限される場合に必要となる構成。

### 要件

| 項目 | 要件 |
|------|------|
| スループット | 1Gbps以上のワイヤレート処理 |

| 管理 | SNMP v3、syslog、NTP |
| 冗長 | VRRP/HSRP（冗長構成時） |

### 候補機種カテゴリ

即座に特定機種を推奨するのは時期尚早。ネットワーク構成の全体像が判明した後に選定するが、カテゴリとして：

| カテゴリ | 例 | 備考 |
|---------|-----|------|
| L3スイッチ（コア兼用） | Catalyst 9300, Aruba CX 6300 | AP選定でArubaを選択した場合、統一管理のメリットあり |
| エンタプライズルーター | YAMAHA RTX1300, Cisco ISR 1000 | YAMAHA RTX1300はRTX3500の後継世代、移行しやすい |
| UTMアプライアンス | FortiGate, Sophos XGS | FW/VPN/IPS統合が必要な場合 |

**AP選定との連携:** AP候補としてAruba（HPE）が有力な場合、上流スイッチ/ルーターもAruba CXシリーズで統一するとAOS-CXによる統合管理・トラブルシューティングが容易になる。ベンダーロックインのリスクはあるが、現状の4社混在カオスに比べれば統一のメリットが圧倒的に上回る。

### 建物の実態: 「12棟」の内訳

名目上は12棟だが、**実質的な建物グループは大きく3つ + KIC**に分類できる。

#### グループ1: 本館棟群（内部通路で接続）— 山側

**①本館・西館・西別館・②中館・北館**が増改築によって物理的に一体化している。本館2Fフロア図面に西館・中館/北館への接続通路が明示されている。

- 棟間のネットワーク接続に外部配線は不要（館内配線で完結）
- ネットワーク設計上は**1つの建物**として扱える
- 本館棟群内の配線は館内のEPS/配管を使用可能（ただし現状は隙間ケーブルで運用している箇所あり）
- **本館内にオンプレミスサーバが存在**する（公開情報の「Linuxサーバ群によるインターネット接続校内LAN環境」の実体と推定）。規模・用途・ラック位置は未確認
- 大昔に**メインフレームが設置されていた部屋**が存在する → 電源容量（高圧受電分岐）・空調（専用PAC）・フリーアクセスフロア等のファシリティが残っている可能性。コアルーター/サーバの設置場所として流用できれば、新規にサーバルームを整備するコストを削減できる
- **フロア構成**: 本館B1〜7F、西館B1〜5F、中館1〜3F、北館1〜3F

**注意:** ③東館は①本館と公道を挟んでおり（配置図で確認）、**公道を横断する架空ファイバ**で接続されているため、グループ1（一体建物）ではなくグループ2（既存ファイバ接続）に分類している。

#### グループ2: 既存架空ファイバで接続済みの棟

**③東館・④北野館・⑤南館**は、本館（グループ1）との間に架空ファイバが敷設されており、現役で使用されている。

**敷設形態**: 以下の3ルートで、各建物の**4階相当の高さに架空ファイバ**が確認されている:
- **①本館 ↔ ⑤南館**（公道横断）
- **①本館 ↔ ③東館**（公道横断）
- **⑤南館 ↔ ④北野館**（公道横断）
- ※③東館 ↔ ④北野館は隣接地のため架空線なし（建物間が近接し、敷地内で接続）

これにより①〜⑤の全棟がファイバで相互接続されている:
```
①本館(G1) ──架空──  ③東館(G2) ──隣接地──  ④北野館(G2)
  │                                            │
  │──架空──  ⑤南館(G2) ──────架空──────────┘
```

**道路占用許可の存在（推定）**: 4階相当の高さで公道を横断する架空ファイバが存在する以上、**道路占用許可が取得済みである可能性が極めて高い**。道路占用許可なしに公道上空に架空ケーブルを渡すことは道路法違反であり、学校法人が無許可で行っているとは考えにくい。これは以下の点で重要:

1. **北野地区でも架空ケーブルの占用許可実績がある** — 景観規制地区であっても架空ファイバの敷設が認められた前例
2. **既存の占用許可を更新・拡張できる可能性** — 同じルートでの新規ファイバ追設や架替えは、新規申請より遥かにハードルが低い
3. **許可の名義・条件が重要な調査事項** — 学校法人名義なのか、NTTや通信事業者名義なのかで、今後の活用方法が変わる
4. **南側グループ（グループ3）への延伸の根拠** — 同じ北野地区内で架空ファイバの前例があれば、不動坂横断の架空敷設も許可される見込みが立つ

その他の確認済み事実:
- ファイバの種別（SM/MM）・敷設時期・管理者が不明
- 北野館MDFに**「至西館」と書かれた光ファイバのパッチパネル**が確認されている → ラベリングがされていたということは、ある時期にはまともに管理されていた。西館は本館棟群の一部であり、北野館↔本館棟群間のファイバ接続の物証
- 現役ではあるが**まともにメンテナンスされている様子はない**
- **OTDRによるヘルスチェックが必要**。測定結果次第でリプレース判断
- 安全側に倒すならリプレースが望ましいが、測定で問題なければ流用も選択肢
- **南館1F「ネットワークラボI・II」** — 名称のみでNW機器はなく、通常授業で使用されている教室。OA床は名残で残っている
- **北野館フロア構成**: B2〜5F（ソニックホール、レコーディングスタジオ等の音響設備あり）
- **南館フロア構成**: B1〜5F（ゲームソフト分野が2〜3Fに集中、各フロア6教室 — 帯域需要が非常に高いエリア）

**既存架空ファイバの調査事項（優先度: 高）:**

| 調査項目 | 目的 | 調査方法 |
|----------|------|---------|
| 道路占用許可の有無と名義 | 許可の更新・拡張の可否を判断 | 事務局への確認 or 神戸市建設事務所への照会 |
| 許可の条件・期限 | 更新時期、追加ケーブルの可否 | 占用許可書の確認 |
| ファイバの種別（SM/MM）と芯数 | 流用可否・CWDM適用可否の判断 | OTDRによる測定 + 両端のコネクタ確認 |
| 敷設時期と施工業者 | 同じ業者に追加工事を依頼できる可能性 | 事務局の契約記録 |
| 架空支持物の状態 | メッセンジャーワイヤー・取付金具の経年劣化 | 目視確認 |
| 建物貫通部の防水処理 | 雨水侵入による劣化リスク | 目視確認 |

→ 本館棟群 + 北野館 + 南館は、既存ファイバが健全であれば**同一ネットワークとして構成可能**。既存の架空ルートと占用許可を活用すれば、ファイバの架替え（48芯SM化）も現実的。**不動坂の北側に位置するため、南側グループとの接続には公道横断が必要**だが、北側での架空実績は南側への延伸の根拠となる

#### グループ3: 不動坂の南側の棟群（遠隔棟）

**⑥MM館・⑦共創館・⑨一宮館・⑩教育研究棟/学生会館**は不動坂の南側に位置する。本館棟群から**約20m以上離れており、かつ不動坂（公道）を挟む**。

- 地下配線やNTTダークファイバが引かれている可能性は否定できないが、**ないものと考えるのが自然**
- 新規の棟間接続が必要。**公道横断が確実に発生**する
- **⑩教育研究棟/学生会館（= KIC所在地、SINET引き込み点）が南側グループに位置** → KICのSINET接続を南側グループの中継とすることで、公道横断の回数を最小化できる可能性
- **共創館**: 6階建て、ゲームソフト/esports（4〜5F）+ グラフィックス（1〜3F,6F）— 帯域需要高
- **MM館**: フォーリースタジオ、映像編集室、デジタルサウンドラボ、マルチメディアスタジオ — サウンド系集中、大容量データ
- **一宮館**: キャリアセンターのみ（1F）— 帯域需要は比較的低い
- **⑧教育管理棟**: 企画広報室、国際交流部。配置図に「大学院」と記載されているが、実際の大学院はここではない

#### ⑩教育研究棟/学生会館（KIC所在地）

**⑩ = KIC（神戸情報大学院大学）の実際の所在地**。SINET6 1Gbpsが引き込まれている建物。

- **SINET6 1Gbps接続済み**
- 南側グループの端に位置
- 南側グループの他の棟（共創館、一宮館、MM館、教育管理棟）に近接 → **SINETの物理接続ポイントとして利用できる可能性**

### ネットワーク設計への影響

| 制約 | 影響 | 備考 |
|------|------|------|
| **不動坂（公道）による南北分断** | グループ1+2（北側）とグループ3（南側）の間に**公道横断が確実に必要**。NTT管路借用 or 道路占用許可が必須 | 配置図で確認済み。一宮交番も不動坂沿い |
| **急傾斜地** | 直線距離が短くても高低差によりケーブル実長が伸びる | 光ファイバの曲げ半径制約にも注意 |
| **北野地区の景観規制** | 北野町は神戸市の都市景観条例の重点地区。ただし**既に本館-南館-東館-北野館間で公道横断の架空ファイバが現存**（4階相当の高さ）。道路占用許可が取得済みと推定され、架替え・追設の余地がある | 既存占用許可の名義・条件の確認が最優先 |
| **KICが南側グループに位置** | SINET引き込みをKIC経由とする場合、南側グループへは近距離で接続可能。**ただし北側グループへは不動坂を越える必要がある** | KIC↔本館棟群間の物理経路が鍵 |
| **南館の位置** | 南館は北側グループと南側グループの中間に位置し、ファイバで本館と接続済み。不動坂の北側にある | — |
| **建物の築年数** | 古い建物は館内の配管（EPS・PS）が狭い or 存在しないケースがある | 本館では天井裏配管があるにもかかわらず隙間ケーブルで運用している実態あり |

- `inter-building-connectivity.md` — 拠点間接続手段の総合比較（専用線・WDM・無線ブリッジ・SINET）

### 棟間配線の設計基準

#### ファイバ種別: シングルモード（SM）を採用

| 項目 | シングルモード（SM） | マルチモード（MM） |
|------|---------------------|-------------------|
| コア径 | 9μm | 50μm / 62.5μm |
| 伝送距離 | 数km〜数十km | 数百m（OM4で550m@10G） |
| トランシーバ価格 | やや高い（LR SFP+: 1〜3万円） | 安い（SR SFP+: 5千〜1万円） |
| 将来性 | 100G/400G対応に有利 | 短距離では100G対応可能 |

20〜100mの短距離ならどちらでも動作するが、**将来の400GbE（400GBASE-LR4等）**へのアップグレードを見据えてSMが有利。SINET6も光ファイバレベルでSMを前提としている。

#### 推奨芯数: 24芯または48芯

- 10GbE/25GbEリンクは2芯（Tx/Rx各1芯）で動作
- 将来のCWDM利用、冗長リンク、監視用芯線を考慮すると48芯が安心
- ケーブル自体の価格差は芯数を増やしても小さい（敷設工事費が支配的）
- **一度管路を通してしまえば芯数追加は容易ではないため、初期に多めに敷設するのが鉄則**

#### 冗長ルートの考え方

コアSWを冗長ペアで設置し、各棟からデュアルホーム接続するスター構成が最も現実的。不動坂を挟む北側/南側の接続は**物理的に異なるルート（東側と西側）で冗長化**すべき。

### 公道横断の配線方法

北野地区は**景観規制地区**（北野町山本通重要伝統的建造物群保存地区、神戸市都市景観形成地域）だが、**既に複数の架空ファイバが公道を横断して敷設されている実績がある**。

#### 架空ケーブル — 既存の前例あり

**既存の架空ファイバ（確認済み）:**
- 本館 ↔ 南館、本館 ↔ 東館、南館 ↔ 北野館の3ルートで、各建物の**4階相当の高さに架空ファイバ**が現存
- これらは公道を横断しており、**道路占用許可が取得済みと推定**される（道路法上、無許可での公道上空への架設は違法）

**景観規制との関係:**
- 兵庫県無電柱化推進計画で北野地区周辺は無電柱化の方向
- トアロード等の近隣で無電柱化（地中化）が完了済み
- 都市景観条例で工作物の建設に届出義務、景観形成基準に不適合なら変更勧告
- 一般論としては新規の架空ケーブル敷設は政策方向に逆行し困難

**ただし、既存の占用許可が存在する場合:**
- **同一ルートでの架替え（張替え）は新規敷設より遥かにハードルが低い** — 既存ケーブルの更新は占用物件の変更届で済む可能性がある
- **追加ケーブルの共架** — 既存の支持物（メッセンジャーワイヤー）に余裕があれば、追加ファイバの共架が可能な場合がある
- **南側（グループ3）への延伸根拠** — 同じ北野地区内で架空ファイバの前例があれば、不動坂横断の架空敷設にも道筋が立つ。既存の占用許可の名義人・条件を確認し、同条件での追加申請が可能か検討する

**結論: 既存前例を活用すべき。** 既存の架空ファイバと占用許可の実態調査が最優先。新規ルート（不動坂横断）についても、既存前例を根拠に神戸市建設事務所に事前相談を行う。地中管路方式と架空方式の両方を選択肢として維持する。

#### 地中管路方式（推奨）

公道横断で光ファイバを通すための方法は4つある:

| 方法 | 概算費用 | 期間 | 備考 |
|------|---------|------|------|
| **NTT既設管路の空き利用** | 50〜200万円 | 3〜6ヶ月 | **最安・最速**。空き管路があることが前提 |
| マイクロトレンチ工法 | 100〜300万円 | 3〜6ヶ月 | 幅1〜5cm・深さ60cmの浅い溝。従来開削の約1/5のコスト |
| 推進工法（非開削） | 300〜800万円 | 6〜12ヶ月 | 道路下にφ100mm管路を推進。交通規制が最小限 |
| 開削工法 | 200〜600万円 | 4〜8ヶ月 | 道路を掘削して管路を埋設 |

**NTT路面溝敷設型光ケーブル**: NTTが開発した新技術。幅8mm・深さ20〜30mmの溝に外径6mmのケーブル（最大40芯）を敷設し、道路補修材で充填。作業時間は従来の1/5。景観への影響がほぼゼロ。

#### 道路占用許可（神戸市）

| 手続き | 内容 |
|--------|------|
| 事前相談 | 所管建設事務所 |
| 申請期限 | 占用予定日の**1ヶ月前**まで |
| 許可取得 | 2〜3週間（警察協議を除く） |
| 占用料（地下電線） | **12円/m/年**（甲地=市街化区域） |
| 占用料（地下埋設管） | 83〜180円/m/年（外径による） |
| 景観規制の届出 | 行為着手の**30日前**までに別途必要 |

占用料自体は安価だが、**管路新設の土木工事費が大きい**。NTTに既設管路の空きがあるケースが最もコスト効率が良い。

### NTTダークファイバの利用可能性

#### 最重要: 学校法人は直接利用不可

NTTダークファイバは**電気通信事業者（総務大臣の登録/届出済み）限定**。学校法人が利用するには:

1. **電気通信事業者経由**: ダークファイバをNTTから借り受けた事業者（エクシオグループ、ネディア等）が「芯線貸し」サービスとして提供
2. **NTT管路の借用も同様に電気通信事業者限定**

#### 接続料金（NTT西日本、2024年4月適用、税抜）

| 項目 | 月額料金（1芯あたり） |
|------|---------------------|
| 加入光ファイバ（シェアドアクセス方式） | **1,796円** |
| 加入光ファイバ（シングルスター方式） | **2,306円** |
| 中継光ファイバ | **1.295円/m** |

上記はNTTが事業者に課す**卸値**。エンドユーザ向け価格はマージン加算後。

#### 北野地区への適用性: 短距離にはオーバースペック

NTTダークファイバは必ず**NTT局を経由**する構成（ユーザビルA → NTT局 → ユーザビルB）になるため、棟間が20〜100mであっても両ビルからNTT局までのファイバ料金がかかる。**棟間直接接続には向かない**。SINETアクセス回線（NTT局→SINETノード）としての利用は有効。

### CWDM（粗密波長多重）— 既存ファイバの帯域増強

既存ファイバや新設ファイバの帯域を、芯数を増やさずに倍増させる技術。

#### CWDMの仕組み

1本の光ファイバ上に異なる波長の光信号を多重して伝送する。パッシブ機器のためMux/Demuxに**電源不要、故障点が少ない**。

| 項目 | CWDM（推奨） | DWDM |
|------|-------------|------|
| チャネル数 | 最大18ch | 40〜96ch以上 |
| 温度制御 | 不要 | 必要 |
| コスト | **低い** | 高い |
| 適用場面 | キャンパスネットワーク、短〜中距離 | 長距離基幹、大規模DC間 |

#### 構成例（棟間接続）

```
 本館MDF                                    共創館MDF
┌────────────┐                            ┌────────────┐
│ L3スイッチ  │                            │ L3スイッチ  │
│  ├─ 10G SFP+ (CWDM 1270nm) ─┐    ┌─ 10G SFP+ (CWDM 1270nm) ─┤
│  ├─ 10G SFP+ (CWDM 1290nm) ─┤    ├─ 10G SFP+ (CWDM 1290nm) ─┤
│  ├─ 10G SFP+ (CWDM 1310nm) ─┤    ├─ 10G SFP+ (CWDM 1310nm) ─┤
│  └─  ...（必要な波長数）   ─┤    ├─  ...                   ─┤
└────────────┘              │    │              └────────────┘
                          ┌──┴────┴──┐
                          │ CWDM     │
                          │ Mux/Demux│ × 2台（両端）
                          └─────┬────┘
                                │
                           1本の光ファイバ（既存 or ダークファイバ）
```

各チャネルに10GbE CWDM SFP+を挿すことで、1芯あたり**最大80Gbps（8ch × 10G）**。将来25G/100G SFPに差し替えれば更に増速可能。

#### 機器コスト（8ch × 10G = 80Gbps、両端分）

| 機器 | 参考価格 |
|------|---------|
| 8ch CWDM Mux/Demux ペア（パッシブ、1U） | $270〜$1,000 |
| 10G CWDM SFP+ × 16個（FS.com互換品） | $480〜$1,280 |
| **合計** | **約15万〜35万円** |

ランニングコストはほぼゼロ。**ダークファイバとの組み合わせが最もコスパが高い**。

#### 事例

- **JANOG41**: ダークファイバ上にCWDMで10Gbps × 2本のインターネット接続を構築
- **エクシオグループ大学事例**: 理工系大学で2km×2ルート + 4km×2ルートのダークファイバ構成、「波長多重化可能な構成で構築し、将来の帯域増にも対応」

| 距離 | 80GHz帯の降雨減衰 | 実用性 |
|------|-------------------|--------|
| 100m | 約0.9dB | 問題なし |
| 500m | 約4.4dB | 許容範囲 |
| 1km | 約8.7dB | 大雨時に帯域低下の可能性 |

### SINET仮想大学LAN — 物理敷設不可時のフォールバック

各棟にSINETアクセス回線を個別に引き、**SINET仮想大学LANで論理的にL2統合**する方法。公道横断の物理工事が一切不要。

```
 棟A                    棟B                    棟C
┌─────┐              ┌─────┐              ┌─────┐
│ L3SW │              │ L3SW │              │ L3SW │
└──┬──┘              └──┬──┘              └──┬──┘
   │                    │                    │
   │ アクセス回線A      │ アクセス回線B      │ アクセス回線C
   │                    │                    │
┌──┴────────────────────┴────────────────────┴──┐
│              SINET6（仮想大学LAN）              │
│         全棟が同一VLANで通信可能              │
└────────────────────────────────────────────────┘
```

**理化学研究所の事例**: 11拠点・500VLANの環境を仮想大学LANで運用。広域イーサネットから移行し、**コスト60%以上削減 + 帯域1Gbps→10Gbps**に向上。

**デメリット**: アクセス回線費用が棟数分かかる。SINET利用規程上、専門学校の一般トラフィック通過の可否は要確認。

### グループ別の接続方針

#### グループ1（①本館・西館・西別館・②中館・北館）→ 棟間配線不要

内部通路で一体化しているため、館内の配線（EPS・天井裏配管）で対応可能。新規の棟間接続工事は不要。

#### グループ2（③東館・④北野館・⑤南館）→ 既存架空ファイバの評価 + CWDM増速

①〜⑤の全棟が4本の架空ファイバ（うち3本は公道横断）で相互接続済み。方針：

1. **OTDRによるヘルスチェックを実施**
2. 測定結果が合格基準を満たす → **既存ファイバを流用 + CWDMで増速**（8ch × 10G = 80Gbps）
3. 測定結果が不合格 or 余裕が不十分 → **リプレース**（48芯SMファイバ、同じ経路に敷設）

既存ファイバはメンテナンスされている様子がないため、安全側に倒すならリプレースが望ましい。ただし測定で問題なければCWDMを載せることで少ない芯数でも十分な帯域を確保できる。

#### グループ3（共創館・一宮館・マルチメディア館等）→ 新規接続

既存の棟間接続がない前提で設計する。全体方針としてパターン1（集約型）とパターン2（分散型）の2案があり（「目標構成」セクション参照）、接続手段の調査結果をもとに以下の選択肢を比較する。

#### 接続手段の総合比較

| 手段 | 帯域 | 月額コスト | 初期コスト | 公道横断 | 10G対応 |
|------|------|----------|----------|---------|---------|
| **ダークファイバ + CWDM** | 80G〜 | 数千〜数万円 | 15万〜50万円 | NTT管路利用（事業者経由） | Yes |
| **60GHz無線ブリッジ** | 最大7.2G | 0円 | 30万〜80万円 | **不要** | No |
| **80GHz無線ブリッジ** | 最大10G | 微少 | 100万〜300万円 | **不要** | Yes |
| **SINET仮想大学LAN** | 10G〜 | 回線数分 | 中 | **不要** | Yes |
| フレッツ+VPNワイド | ベストエフォート | 約1.5万円 | 低 | 事業者工事 | ベストエフォート |

### 光ファイバ品質検証（グループ2: 既存ファイバ）

OTDRによる測定で既存ファイバの流用可否を判断する。

| 測定項目 | 合格基準（目安） |
|---------|--------------|
| 挿入損失 | 1G: < 2.0 dB/km, 10G: < 0.5 dB/km |
| コネクタ損失 | < 0.3 dB/接点 |
| 曲げ損失 | 異常な減衰ポイントがないこと |
| 反射減衰量 | > 35 dB |

**測定リソース:** OTDR測定器は高額（数十万〜数百万円）。選択肢：
- 通信工事業者への委託（回線工事と合わせて依頼）
- KIC教授経由で大学の測定器を借用（可能性は低いが確認の価値あり）

## 実施順序

```
[Phase 0] 現状調査（ここが最優先）
  ├── ISP契約の棚卸し（事務局・経理）
  ├── 全棟のルーター・MDF調査（現地）
  ├── 棟間接続の確認（traceroute + 現地）
  ├── IP体系の全体像把握
  ├── SINET利用可能性の確認（KIC教授→NII）
  ├── NTTダークファイバ照会（本館→各遠隔棟ルートの提供可否・コスト）
  └── 既存ファイバ（本館↔北野館・南館）のOTDR測定

[Phase 1] 回線集約の設計
  ├── 上流回線の決定: SINET / 商用ISP / ハイブリッド
  ├── 遠隔棟接続の決定: パターン1（集約型）/ パターン2（分散型）
  │   └── NTTダークファイバ照会の結果に依存
  ├── コアルーター/L3スイッチの選定（AP選定と連携）
  └── IPアドレス体系の再設計

[Phase 2] 回線切替・ルーターリプレース
  ├── 新回線の開通（本館主回線 + 遠隔棟の接続手段）
  ├── コアルーターの設置・設定（本館MDF）
  ├── グループ2: 既存ファイバ流用 or リプレース（OTDR結果に依存）
  ├── グループ3: ダークファイバ開通 or 個別回線+OSPF/BGP構築
  └── 旧回線の段階的廃止

[Phase 3] 旧回線・旧ルーターの撤去
  ├── 全棟の切替完了確認
  ├── 旧ISP契約の解約
  └── 旧機器の撤去
```

**注意:** Phase 0の現状調査は**L8（組織政治）を経由しなくても一部は実施可能**。tracerouteやnmapによるリモート調査は学生の立場でも技術的には可能（ただし校内ネットワークのスキャンは許可の範囲を要確認）。ISP契約情報の取得には事務局の協力が必要であり、ここがL8の最初の接触点になる。

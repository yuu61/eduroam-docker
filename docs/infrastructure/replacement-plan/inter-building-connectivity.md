# 拠点間接続手段

## 背景

神戸電子専門学校（北野地区）のキャンパスは複数棟が公道を挟んで分散しており、棟間を10Gbps以上で接続する必要がある。本文書では、短距離（20m〜数km）の拠点間接続に利用可能な手段を網羅的に整理する。

関連文書:
- `upstream-connectivity.md` -- 上流回線リプレースプラン（帯域試算、リプレース方針）
- `dark-fiber-investigation.md` -- NTTダークファイバの制度・利用可能性調査

## 1. 広域イーサネット / イーサネット専用線

### 1.1 サービス概要

拠点間をL2（イーサネットフレーム）で接続する通信事業者提供のマネージドサービス。両端に回線終端装置（CPE）を設置し、事業者のネットワーク経由で拠点間を接続する。

| 特性 | 広域イーサネット | イーサネット専用線 |
|------|---------------|-----------------|
| 接続形態 | 多対多（メッシュ） | 1対1（Point-to-Point） |
| 帯域 | 共有型（QoS制御あり） | 帯域保証型 |
| 拡張性 | 拠点追加が容易 | 拠点追加ごとに回線追加 |
| 用途 | 多拠点VPN | DC間接続、バックボーン |

### 1.2 主要サービスと10Gbps料金

#### NTT西日本 ビジネスイーサ ワイド（広域イーサネット）

- **提供帯域**: 10Mbps〜10Gbps
- **月額料金**: 拠点分布・規模・利用形態により個別見積もり（非公開）
- **特徴**: QoS制御、フィルタリング、サブグループ設定、トラヒックレポート、NTT東西間相互接続
- **長期割引**: 3年 7%、6年 11%
- **参考**: 関連サービス「高速広帯域アクセスサービス」10Gbps タイプ1は月額約99万〜198万円（NTT東日本参考値、距離依存）

#### KDDI 国内イーサネット専用サービス（帯域保証型 10Gbps）

- **月額料金（税抜）**: 距離に応じて以下の範囲
  - 同一エリア内（Dual Class）: 約204.6万円/月〜
  - 同一エリア内（Single Class）: 約325.6万円/月〜
  - 50km以内: 約799万〜1,074万円/月
- **端末装置**: 13.2万円 × 2拠点
- **初期費用**: 回線接続工事 2,750円 + 端末装置設置 22,000円/拠点
- **最低利用期間**: 1年
- **長期割引**: 3年 7%、6年 11%

#### オプテージ（関西電力系） イーサ専用線

- **提供エリア**: 関西2府5県（神戸は提供エリア内）
- **月額料金（税抜）**: 距離依存
  - 15km以内: 268万円/月
  - 30km以内: 738万円/月
- **初期費用**: 回線接続工事 2,500円 + 端末装置取付 20,000円/片端
- **最低利用期間**: 1年
- **長期割引**: 3年 7%、6年 11%

#### エネコム 高速イーサネット専用線

- **月額料金（税抜）**: 288万円/月（端末回線1回線ごと）

### 1.3 同一局舎エリア内での特性

北野地区の建物群は20m〜数百mの範囲に集中しており、おそらく同一NTT収容局の加入者区域内に収まる。この場合:

- **距離加算が最小**: 専用線料金の距離加算部分がほぼゼロに近くなる
- **中継系設備が不要**: 加入者系設備のみで完結するため、料金・敷設工期ともに有利
- **ただし10Gbps帯域保証は高額**: 同一エリアでも10Gbps帯域保証型は月額200万円〜が一般的

### 1.4 評価

| 項目 | 評価 |
|------|------|
| 帯域 | 10Gbps帯域保証可能 |
| 信頼性 | SLA付き、事業者管理 |
| 初期費用 | 低い（装置設置のみ） |
| 月額費用 | **非常に高い**（10Gbpsで月200万円〜） |
| 適合性 | 短距離の棟間接続には過剰にコスト高。上流回線（ISP/SINET接続）向き |

**結論**: 棟間接続（20m〜100m）にイーサネット専用線を使うのはコスト的に非現実的。上流のISP/SINET接続回線としてなら検討に値する。

## 2. ダークファイバ

詳細は `dark-fiber-investigation.md` を参照。ここでは概要のみ記載。

### 2.1 NTTダークファイバ

- **利用条件**: 電気通信事業者限定（学校法人は直接利用不可）
- **間接利用方法**: ダークファイバ事業者（エクシオグループ等）経由で借りる
- **接続料**: 中継系は距離比例（約1.083円/m/月 + 管理費40円/芯）
  - 例: 100mの場合 → 約148円/芯/月（極めて安価）
  - 例: 1kmの場合 → 約1,123円/芯/月
  - 例: 5kmの場合 → 約5,455円/芯/月
- **加入者系**: 別途料金体系（ラストワンマイル向け）
- **注意**: 接続料はNTTが事業者に課す卸値。エンドユーザ向け料金はこれに事業者のマージン・工事費等が加算される

### 2.2 自営光ファイバ敷設

公道を横断する場合:
- **道路占用許可**: 道路管理者（神戸市）への申請が必要
- **電気通信事業者の場合**: 義務占用（一定基準を満たせば許可される）
- **非事業者の場合**: 公益事業特権の適用なし。占用許可取得は困難な場合がある
- **簡易敷設**: NTTが路面溝に敷設可能な光ケーブル技術を開発（将来的な選択肢）

### 2.3 評価

| 項目 | 評価 |
|------|------|
| 帯域 | 芯線上に載せる機器次第（10G/40G/100G/400G） |
| 月額費用 | 非常に安い（NTT卸値ベースで数千円/芯/月） |
| 初期費用 | 中程度（敷設工事、事業者手配） |
| 手続き | 電気通信事業者経由での手配が必要 |
| 適合性 | 短距離棟間接続に最適。WDMとの組み合わせで更に効率的 |

## 3. WDM（波長分割多重）

### 3.1 技術概要

1本の光ファイバ上に、異なる波長の光信号を多重して伝送する技術。既存の光ファイバインフラを活かしたまま、帯域を何倍にも増強できる。

### 3.2 CWDM vs DWDM

| 項目 | CWDM（粗密波長多重） | DWDM（高密波長多重） |
|------|---------------------|---------------------|
| 波長間隔 | 20nm | 0.8nm / 0.4nm / 0.2nm |
| チャネル数 | 最大18ch（1270〜1610nm） | 40〜96ch以上（C帯 + L帯） |
| 最大距離 | 約80km | 数百km〜数千km（アンプ込み） |
| 温度制御 | 不要（冷却レーザ不要） | 必要（冷却レーザ必須） |
| 電源 | パッシブMux/Demuxは不要 | アンプ使用時は必要 |
| コスト | **低い** | 高い |
| 適用場面 | キャンパスネットワーク、メトロ、短〜中距離 | 長距離基幹、大規模DC間 |

### 3.3 短距離キャンパスネットワークでの推奨: CWDM

北野地区の棟間接続（20m〜数百m）には**CWDMが最適**。理由:

1. **距離が短い**: CWDMの80km制限は余裕
2. **チャネル数が十分**: 8〜18chで棟間接続には十分すぎる帯域
3. **パッシブ動作**: Mux/Demuxに電源不要、故障点が少ない
4. **低コスト**: DWDMの1/3〜1/5程度の機器コスト

### 3.4 CWDM構成例（棟間接続）

```
 本館MDF                                    共創館MDF
┌────────────┐                            ┌────────────┐
│ L3スイッチ  │                            │ L3スイッチ  │
│  ├─ 10G SFP+ (CWDM 1270nm) ─┐    ┌─ 10G SFP+ (CWDM 1270nm) ─┤
│  ├─ 10G SFP+ (CWDM 1290nm) ─┤    ├─ 10G SFP+ (CWDM 1290nm) ─┤
│  ├─ 10G SFP+ (CWDM 1310nm) ─┤    ├─ 10G SFP+ (CWDM 1310nm) ─┤
│  └─  ...（必要な波長数）   ─┤    ├─  ...                   ─┤
└────────────┘              │    │              └────────────┘
                          ┌──┴────┴──┐
                          │ CWDM     │
                          │ Mux/Demux│ × 2台（両端）
                          └─────┬────┘
                                │
                           1本の光ファイバ（既存 or ダークファイバ）
```

- 各チャネルに10GbE CWDM SFP+を挿すことで、1芯あたり最大180Gbps（18ch × 10G）
- 実用的には8chで80Gbps（片方向）が現実的
- 将来25G/100G CWDM SFP28/QSFPに差し替えれば更に増速可能

### 3.5 機器コスト参考値

| 機器 | 参考価格（USD） | 備考 |
|------|---------------|------|
| 8ch CWDM Mux/Demux（パッシブ、1U） | $135〜$500 | FS.com等で入手可能。ペアで $270〜$1,000 |
| 10G CWDM SFP+ トランシーバ | $30〜$80/個 | FS.com互換品。8ch × 2端 = 16個で $480〜$1,280 |
| 18ch CWDM Mux/Demux | $300〜$800 | より大容量が必要な場合 |

**CWDMの総コスト例（8ch × 10G = 80Gbps、両端分）**:
- Mux/Demux ペア: 約$500〜$1,000
- SFP+ 16個: 約$480〜$1,280
- **合計: 約$1,000〜$2,300（約15万〜35万円）** ← ランニングコストほぼゼロ

### 3.6 大学・研究機関での事例

- **JANOG41**: 会場内ネットワークでCWDM装置を使用し、ダークファイバ上に10Gbps × 2本のインターネット接続を構築
- **理化学研究所**: 11拠点・500VLANの環境をSINET仮想大学LANで運用。以前は広域イーサネットだったが、60%以上のコスト削減と1Gbps→10Gbpsの帯域増強を実現
- **エクシオグループの大学事例**: 理工系大学で2km×2ルート + 4km×2ルートのダークファイバ構成。「波長多重化可能な構成で構築し、将来の帯域増にも対応」

### 3.7 評価

| 項目 | 評価 |
|------|------|
| 帯域 | 80Gbps〜（CWDM 8ch × 10G）。将来拡張も容易 |
| 月額費用 | ほぼゼロ（パッシブ機器、電源不要） |
| 初期費用 | 15万〜35万円程度（機器のみ）。光ファイバは別途 |
| 運用性 | パッシブ故障点なし、高信頼 |
| 前提条件 | 棟間の光ファイバ（既存 or ダークファイバ）が必要 |
| 適合性 | **ダークファイバとの組み合わせで最強のコスパ** |

## 4. 無線ブリッジ（ミリ波）

### 4.1 技術概要

ミリ波（30〜300GHz）帯の電波で拠点間をPoint-to-Pointで接続する無線ブリッジ。光ファイバの敷設が困難な場合の代替手段。

### 4.2 60GHz帯（V-band）

| 項目 | 内容 |
|------|------|
| 周波数 | 57〜66GHz |
| 日本での規制 | **免許不要**（特定小電力無線局、2000年制度化） |
| 最大スループット | 2〜7.2Gbps（チャネルボンディング時） |
| 最大距離 | 200m〜1km（製品により異なる） |
| 酸素吸収 | 60GHz付近で顕著（約15dB/km）→ 長距離不利 |

#### 主要製品

| 製品 | メーカー | スループット | 距離 | 特徴 |
|------|---------|------------|------|------|
| cnWave V1000 | Cambium/ハイテクインター | 最大2Gbps | 200m | 小型、IP66/67、−40〜+60℃ |
| cnWave V3000 | Cambium/ハイテクインター | 最大7.2Gbps | 1km | 高指向性、SFP+対応 |
| cnWave V5000 | Cambium/ハイテクインター | 親機 | — | 1対多（V1000/V3000と組合せ） |
| EtherHaul-600TX | Siklu/HCNet | — | — | 60GHz PtP |
| airFiber 60 HD | Ubiquiti | 数Gbps | 2km | SFP+対応、技適状況要確認 |
| airFiber 60 XG | Ubiquiti | 最大6Gbps | 4km+ | SFP+対応、技適状況要確認 |

**注意**: Ubiquiti製品の日本での技適取得状況は要確認。日本国内でAmazon.co.jpでの販売は確認されるが、屋外使用での技適適合性は個別に確認が必要。

### 4.3 70/80GHz帯（E-band）

| 項目 | 内容 |
|------|------|
| 周波数 | 71〜76GHz / 81〜86GHz |
| 日本での規制 | **免許（登録）が必要**。移動局登録として総合通信局に申請 |
| 必要資格 | 第三級陸上特殊無線技士 |
| 最大スループット | 最大10Gbps Full Duplex |
| 最大距離 | 3〜5km（降雨条件依存） |
| 制度 | 日本では比較的最近使用可能になった（80GHz帯の制度改正） |
| 利用用途制限 | なし（登録エリア内で設置場所変更可能） |

#### 主要製品

| 製品 | メーカー | スループット | 距離 | 特徴 |
|------|---------|------------|------|------|
| EtherHaul-8010FX | Siklu | 最大10Gbps FD | 〜3km | IP67、SFP+/10GBase-T、VLAN/QoS |
| CableFree MMW E-band | CableFree | 最大10〜40Gbps | 8〜20km | 2+0構成で20Gbps FD |

#### 日本での取扱い

Siklu製品は以下の日本企業が取り扱い:
- エイチ・シー・ネットワークス株式会社（HCNet）
- ソネット株式会社
- 株式会社理経
- 株式会社昭電

### 4.4 降雨減衰の影響

神戸は海沿いに位置し、降雨・霧の影響を受けやすい。ミリ波帯の降雨減衰は重要な考慮事項。

| 周波数帯 | 降雨減衰（概算） | 備考 |
|---------|----------------|------|
| 60GHz | 酸素吸収 約15dB/km + 降雨減衰 | 短距離（200m以下）なら影響軽微 |
| 80GHz | 降雨減衰 約8.7dB/km（99.99%可用性時、ITU-R P.838-3） | 大雨時に帯域低下 |

#### 距離別の降雨影響（80GHz、99.99%可用性目標）

| 距離 | 降雨減衰（概算） | 実用性 |
|------|----------------|--------|
| 100m | 約0.9dB | 問題なし |
| 500m | 約4.4dB | 許容範囲。リンクバジェットに余裕を持たせる |
| 1km | 約8.7dB | 大雨時に帯域低下の可能性。適応変調で対応 |
| 3km | 約26dB | 大雨時に通信断の可能性。冗長回線が必要 |

**Siklu EH-8010FXの降雨対策**: ヒットレス/エラーレス適応帯域・符号化・変調調整機能を搭載。降雨時は自動的に変調方式を下げて帯域を低減しつつ接続を維持する。

#### 神戸での適用性

- 北野地区の棟間距離（20m〜100m）では降雨減衰は**ほぼ無視できる**
- 100m未満なら60GHz帯でも80GHz帯でも99.99%以上の可用性が見込める
- ただし、霧（海沿い特有）は降雨とは異なる減衰特性を持つ。粒径が小さいため高周波ほど影響が小さくなる傾向

### 4.5 費用

| 項目 | 60GHz帯（V-band） | 80GHz帯（E-band） |
|------|-------------------|-------------------|
| 機器費用（ペア） | 約30万〜80万円 | 約100万〜300万円（推定） |
| 免許費用 | 不要 | 電波利用料（年額数千円〜） |
| 免許申請手続き | 不要 | 総合通信局への登録申請 |
| 必要資格 | なし | 第三級陸上特殊無線技士 |
| 工事費 | 設置・アライメント | 設置・アライメント + 免許申請代行 |

### 4.6 評価

| 項目 | 60GHz帯 | 80GHz帯（E-band） |
|------|---------|-------------------|
| 帯域 | 最大7.2Gbps | 最大10Gbps FD |
| 月額費用 | ゼロ | 電波利用料のみ（微少） |
| 初期費用 | 中程度（30万〜80万円） | 高い（100万〜300万円） |
| 免許 | **不要** | 必要 |
| 降雨耐性 | 酸素吸収あり、短距離向き | 降雨減衰あるが適応変調で緩和 |
| 10Gbps到達 | 不可（最大7.2Gbps） | 可能 |
| 適合性 | 100m以下の棟間接続で有力 | 10Gbps必須の場合の選択肢 |

**60GHz帯の限界**: 10Gbpsに届かない（最大7.2Gbps）。ただし棟間接続であれば7.2Gbpsで十分な場合も多い。

## 5. SINET接続校のアクセス回線構成

### 5.1 SINET6の概要

| 項目 | 内容 |
|------|------|
| 運用開始 | 2022年4月 |
| ノード間帯域 | 400Gbps |
| ノード数 | 全国70箇所（各都道府県にDCを設置） |
| アクセス回線速度 | 400G / 100G / 40G / 10G / 1G以下 |
| 加入機関数 | 約1,000機関 |
| 平均アクセス速度 | 15.4Gbps（2023年10月時点） |
| 利用料金 | SINETのサービス利用自体は無料。アクセス回線費用は加入機関負担 |

### 5.2 接続インタフェース

| インタフェース | コネクタ | 用途 |
|--------------|---------|------|
| 1000BASE-T | RJ45 | 小規模機関 |
| 1000BASE-LX | SC (SM) | 小規模機関 |
| 10GBASE-LR | SC (SM) | 標準（最多） |
| 40GBASE-LR4 | SC (SM) | 中〜大規模 |
| 100GBASE-LR4 | SC (SM) | 大規模 |
| 400GBASE-FR4 | SC (SM) | 最大規模 |

### 5.3 アクセス回線の種類

SINETノードへの接続には以下のアクセス回線が利用可能:

| 種類 | 概要 | コスト感 |
|------|------|---------|
| **ダークファイバ** | NTTダークファイバ（事業者経由）で直接ノードに接続。最も高速・低コスト | **最安** |
| **商用帯域保証型専用線** | NTT/KDDI/オプテージ等の帯域保証型イーサネット専用線 | 高い |
| **フレッツ + VPN** | フレッツ光 + L2トンネリング装置で疑似専用線を構成 | 安い（帯域非保証） |
| **UNO（広域LAN）** | SINETが提供する低コスト接続オプション（帯域制限あり） | 安い |
| **他機関経由** | 既にSINET接続済みの機関を経由して間接接続 | 機関間交渉 |

### 5.4 共同調達

NIIは加入機関と共同でアクセス回線の調達を行い、ボリュームディスカウントによるコスト削減を実現している。ダークファイバの共同調達が主な対象。

### 5.5 SINET仮想大学LANサービス

公道で分断されたキャンパスを持つ機関にとって特に有用なサービス:

- SINETに接続した複数キャンパス間で、L2レベルで組織内LANを拡張
- 加入機関がVLAN IDの範囲を指定し、SINET側が自動認識して拠点間通信を実現
- L2VPNサービスよりも柔軟な運用が可能

**理化学研究所の事例**:
- 11拠点・500VLANの環境を仮想大学LANで運用
- 以前の広域イーサネットから移行し、**コスト60%以上削減**
- 帯域が1Gbps → 10Gbpsに向上
- 移行期間: 約1年（2015年4月準備開始 → 12月完了）

### 5.6 兵庫県のSINETノード

- 兵庫県にSINET6のDCノードが設置されている
- DC所在地は非公開（セキュリティ上の理由）。接続検討時にSINETに住所問合せ申請が必要
- 大阪ノードも近接しており、アクセス回線の経路選択肢がある

### 5.7 KIC（神戸情報大学院大学）の現状

- SINET6に**1Gbps**で加入済み
- 同一法人（学校法人コンピュータ総合学園）として、神戸電子をSINETに収容できる可能性がある
- SINETの仮想大学LANを使えば、KIC↔神戸電子の棟間接続をSINET経由で実現することも理論上は可能

### 5.8 群馬県の大学での導入事例

- 商用専用線からダークファイバ利用に乗り換え
- 通信速度: 1Gbps → 10Gbps近くまで向上
- ランニングコスト: 削減
- 接続方式: ダークファイバ + 10GBASE-LR でSINETノードに直結

## 6. 公道横断が必要なケースでの接続手段

### 6.1 日本の法制度

公道を横断して光ファイバを敷設する場合の法的枠組み:

| 方法 | 概要 | ハードル |
|------|------|---------|
| **道路占用許可** | 道路管理者（市区町村 or 国交省）に申請。電柱・管路への添架 | 電気通信事業者は義務占用（比較的容易）。非事業者は個別審査 |
| **NTT管路利用** | NTTの既設管路に光ケーブルを通す（電気通信事業者のみ） | 空き管路の有無による |
| **共同溝** | 道路下の共同溝を利用 | 存在する場所が限定的 |
| **河川・道路管理用光ファイバ開放** | 国交省が敷設した管理用ファイバの余剰芯線を民間に開放 | 1芯から利用可能だが対象区間が限定的 |
| **架空横断** | 電柱間に光ケーブルを架設 | 道路占用許可 + 電柱利用許可。景観規制地区は困難な場合あり |
| **無線ブリッジ** | 物理敷設不要。屋上間でPtP接続 | 許認可不要（60GHz帯の場合） |

### 6.2 北野地区の特殊事情

神戸市北野地区は**北野異人館街に隣接する景観保全地区**であり、以下の制約が想定される:

- 架空配線（空中ケーブル）に景観上の制約がある可能性
- 電柱の新設は困難な可能性
- 地下管路の新設工事にも景観・文化財保護上の制限がありうる

→ このような環境では、**無線ブリッジが最も導入障壁が低い**

### 6.3 大学キャンパスでの一般的な対応方法

日本の大学で公道をまたぐ棟間接続に使われている手段:

| 手段 | 事例・概要 |
|------|-----------|
| **NTT管路 + ダークファイバ** | 最も一般的。NTTの既存管路にファイバを通す |
| **自営管路** | 大規模大学が道路占用許可を取得して独自に管路敷設 |
| **商用専用線** | 広域イーサネット等のマネージドサービスを利用 |
| **SINET仮想大学LAN** | 各棟にSINET接続し、SINET経由でL2接続（理研の事例） |
| **無線ブリッジ** | 光敷設が困難な場合のバックアップ |

### 6.4 SINET仮想大学LANによる棟間接続パターン

公道横断が物理的に困難な場合、各棟に独立してSINETアクセス回線を引き、仮想大学LANで論理的に統合する方法がある:

```
 棟A                    棟B                    棟C
┌─────┐              ┌─────┐              ┌─────┐
│ L3SW │              │ L3SW │              │ L3SW │
└──┬──┘              └──┬──┘              └──┬──┘
   │                    │                    │
   │ アクセス回線A      │ アクセス回線B      │ アクセス回線C
   │                    │                    │
┌──┴────────────────────┴────────────────────┴──┐
│              SINET6（仮想大学LAN）              │
│         全棟が同一VLANで通信可能              │
└────────────────────────────────────────────────┘
```

**メリット**: 公道横断の物理工事が一切不要
**デメリット**: 棟数分のアクセス回線費用が発生。SINET経由のため遅延はダークファイバ直結より大きい

## 7. 接続手段の比較と推奨

### 7.1 総合比較

| 手段 | 帯域 | 月額コスト | 初期コスト | 公道横断 | 信頼性 | 10Gbps対応 |
|------|------|----------|----------|---------|--------|-----------|
| イーサネット専用線（10G） | 10G保証 | 200万〜/月 | 低 | 事業者管理 | 高（SLA） | Yes |
| ダークファイバ + CWDM | 80G〜 | 数千〜数万円/月 | 中（15万〜50万） | NTT管路利用 | 高 | Yes |
| ダークファイバ（WDMなし） | 10G | 数千円/月 | 低 | NTT管路利用 | 高 | Yes |
| 60GHz無線ブリッジ | 最大7.2G | 0円 | 30万〜80万 | **不要** | 中（降雨時低下） | No |
| 80GHz無線ブリッジ | 最大10G | 微少 | 100万〜300万 | **不要** | 中（降雨時低下） | Yes |
| SINET仮想大学LAN | 10G〜 | 回線数分 | 中 | **不要** | 高 | Yes |

### 7.2 推奨構成

#### 第1候補: ダークファイバ + CWDM（本館集約型）

`upstream-connectivity.md` のパターン1（集約型）に対応。

- 本館にコアルーター + SINET/ISP接続
- 本館→各遠隔棟をNTTダークファイバ（事業者経由）で接続
- 各芯線にCWDM Mux/Demuxを設置し、10G × 複数chで帯域確保
- **月額コスト**: ダークファイバ数千円/芯 × 棟数 ≒ **数万円/月**
- **初期コスト**: CWDM機器 + SFP+ = **15万〜35万円/棟ペア**
- **帯域**: 棟あたり80Gbps〜（CWDM 8ch × 10G）

**前提条件**: NTTダークファイバ（事業者経由）が本館→各遠隔棟で利用可能であること

#### 第2候補: 60GHz無線ブリッジ（短距離棟間）

ダークファイバが利用困難な棟間の代替。

- cnWave V3000（最大7.2Gbps、1km）を棟間PtP配置
- 免許不要、工事も屋上/壁面への設置のみ
- 景観保全地区でも電波は規制対象外（建築許可等は別途確認）
- **降雨時**: 100m以下なら99.99%以上の可用性
- **コスト**: 1リンクあたり30万〜80万円（初期のみ）

**限界**: 10Gbpsに届かない。バックボーンではなく、中継用途向き

#### 第3候補: SINET仮想大学LAN（各棟独立接続）

物理敷設が一切不可能な場合のフォールバック。

- KICのSINET接続を法人全体に拡張し、各棟にアクセス回線
- 仮想大学LANで全棟を同一L2ドメインに統合
- **メリット**: 公道横断の物理工事ゼロ
- **デメリット**: アクセス回線費用が棟数倍。SINET利用規程の制約確認が必要

### 7.3 ハイブリッド推奨構成

現実的には複数手段の組み合わせが最適:

```
                      ┌───────────────┐
                      │ SINET / ISP   │
                      └───────┬───────┘
                              │ 10G〜
                       ┌──────┴──────┐
                       │  本館コア    │
                       │  L3スイッチ  │
                       └──────┬──────┘
                              │
         ┌────────────────────┼────────────────────┐
         │                    │                    │
    [既存架空ファイバ]    [ダークファイバ]      [60GHz無線]
    ①本館↔③東館(公道横断) 本館-共創館          本館-一宮館
    ①本館↔⑤南館(公道横断) 本館-MM館           （ダークファイバ
    ⑤南館↔④北野館(公道横断)(CWDM 8ch×10G)      不可の場合）
    ③東館↔④北野館(隣接地)
```

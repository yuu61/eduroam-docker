# eduroam JP 申請システムについて

出典: [NII学術認証推進室 Wiki](https://nii-auth.atlassian.net/wiki/spaces/NIIninsho/pages/44270008)
取得日: 2026-03-16

## 1. 概要

eduroam JP申請システムは、学術無線LANローミング基盤サービス「eduroam JP」における加入・変更・脱退申請を処理するプラットフォーム。代理認証システムおよび認証連携IDサービスの個別申請にも対応している。

**システムURL**: https://office.eduroam.jp/

## 2. ログイン方法

本システムは学認SP として機能するため、学認参加機関では自機関のIdPとの認証連携設定が必須。

**学認非参加機関の場合**: [Orthros](https://ui.orthros.gakunin.nii.ac.jp/) でアカウントを作成してログイン可能。申請システムのリストから「Orthros」を選択する。

**重要**: 加入申請の承認後は、機関責任者または技術担当者としてePPNが登録されているユーザのみがアクセス可能となる。申請者がこれらの担当者でない場合は、必ず担当者のePPNを事前に登録すること。

## 3. 主要機能

- 加入・変更・脱退申請
- 基地局マップのアップロード機能
- RADIUSサーバ設定ファイル生成機能

## 4. SINET加入機関の場合

機関長の押印・郵送が不要。ただしSINET担当者（LAN管理責任者または事務担当者）へ電子メールにて確認が取られるため、事前調整が必要。eduroam JP責任者とSINET LAN管理責任者が同一人物の場合、確認がスキップされる。

## 5. SINET非加入機関の場合

- **加入申請時**: 機関長の記名・押印と郵送が必須
- **変更・脱退申請時**: 機関責任者のePPN登録がある場合、オンラインチェック機能により書面手続きを省略可能

## 6. 代理認証・認証連携IDサービス

独立した利用/利用停止申請が可能で、申請書の提出は不要。

## 7. 申請時の注意点

- 既存加入機関の継続申請も新規申請扱いとなる
- 可能な限り機関職員による申請を推奨（業務委託者の技術担当者配置は許可される）
- 申請前に「申請のポイント」（application-points.md）を参照すること

## 8. IdPの属性設定

eduroam JP申請システム（entityID: `https://office.eduroam.jp/shibboleth-sp`）は **eduPersonPrincipalName（ePPN）の送信を必須** とする。

学認参加機関の場合、attribute-filter.xml にてこのSPに対するePPN送信の設定が必要。

## 9. マニュアル

詳細な操作手順は、公式マニュアル『eduroam JP申請システム操作マニュアル 機関責任者・技術担当者用』を参照。

## 11. 本プロジェクトへの適用メモ

- 学認非参加のため、**Orthros** でアカウント作成してログインする
- 神戸情報大学院大学は**SINET6加入済み**（神戸電子専門学校は未加入）。学校法人名義での申請がSINET加入機関として扱われれば、機関長の押印・郵送を省略可能。扱いはNIIへ要確認
- 技術担当者2名のePPN登録が必要（申請承認後のシステムアクセスに必須）
- **eduroam JP申請システム**: 学認不要 → Orthrosでログイン可能（本プロジェクトはこのルート）
- **認証連携IDサービス**: 学認必要 → 本プロジェクトは自前RADIUS方針のため利用しない
- 将来的にUPKI証明書を利用する場合も、学認参加は不要（UPKI証明書はeduroamとは別の申請経路）

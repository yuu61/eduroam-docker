# eduroam JP 申請資格・参加条件 調査結果

調査日: 2026-03-16（再調査: 学校法人全体を考慮）

## 1. eduroam JP の運営体制

eduroam JPは**国立情報学研究所（NII）**が運営する、教育・研究機関のキャンパス無線LANの相互利用を実現する国際的なネットワークローミングサービス。世界100以上の地域が参加し、国内では**478機関**が参加している（2026年3月時点）。

## 2. 学校法人コンピュータ総合学園の構成

| 項目 | 内容 |
|------|------|
| 法人名 | 学校法人コンピュータ総合学園 |
| 設置校1 | **神戸情報大学院大学**（専門職大学院 / 学校教育法第83条の大学） |
| 設置校2 | **神戸電子専門学校**（専修学校専門課程） |

- 神戸情報大学院大学は「大学」に該当し、eduroam JP本体の参加対象に直接合致する
- 神戸電子専門学校は「専修学校」であり、eduroam JP本体の主たる対象（大学等高等教育機関）には直接該当しない
- ただし、同一法人が大学を設置しているため、**法人全体として eduroam JP本体に参加する根拠がある**

## 3. 参加資格の分析

### 3.1 eduroam JP 本体（NII運用）

対象: **国内の高等教育研究機関等**（大学、短期大学、高等専門学校、研究機関）

参加機関の内訳:
| カテゴリ | 機関数（概算） |
|---------|--------------|
| 国立大学 | 約50 |
| 公立大学 | 約40 |
| 私立大学 | 約180以上 |
| 高等専門学校 | 約50 |
| 研究機関・その他 | 約30 |

#### 学校法人名義での参加実績

eduroam JP参加機関一覧（https://eduroam.jp/node/375 ）から確認した学校法人名義の参加例:

| 機関名 | 所在地 | 備考 |
|--------|--------|------|
| 学校法人三幸学園 | 東京都文京区 | **大学（東京未来大学）＋多数の専門学校を設置する法人** |
| 学校法人北海道科学大学 | 札幌市 | 大学を設置する法人 |
| 学校法人東京国際大学 | 川越市 | 大学を設置する法人 |
| 学校法人東北工業大学 | 仙台市 | 大学を設置する法人 |
| 学校法人茨城キリスト教学園 | 日立市 | 大学を設置する法人 |
| 学校法人日本福祉大学 | 美浜町 | 大学を設置する法人 |
| 学校法人鹿児島純心女子学園 | 鹿児島市 | 大学を設置する法人 |

**特に学校法人三幸学園は、大学＋多数の専門学校（専修学校）を設置する法人が法人名義でeduroam JP本体に参加している先例であり、学校法人コンピュータ総合学園の状況と構造が類似する。**

#### 大学院大学の参加実績

大学院大学としてeduroam JP本体に参加している機関:

- 北陸先端科学技術大学院大学（石川県能美市）
- 奈良先端科学技術大学院大学（奈良県生駒市）

神戸情報大学院大学は専門職大学院であり、学校教育法第83条に定める「大学」に該当するため、eduroam JP本体の参加資格を満たす。

#### 附属校の一括管理

eduroam JP公式サイトには以下の記載がある:

> 大学附属校については、大学側でアカウントを一括して管理することでeduroamをご利用いただけますが、これについては大学としての加入申請に含まれるため、別途学校単位で加入申請をご提出いただく必要はありません。

この規定に基づき、神戸情報大学院大学（大学）の加入申請に神戸電子専門学校を含めて一括管理する形が取れる可能性がある。ただし、神戸電子専門学校は厳密には「附属校」ではなく同一法人の別の設置校であるため、この適用可否はNIIへの確認が必要。

### 3.2 学校向けeduroam（NGHSIG運用）— 代替ルート

初等・中等教育機関や専修学校を対象とした別ルート。eduroam JP本体経由が不可の場合の代替手段。

**運営**: セキュア公衆無線LANローミング研究会（NGHSIG）
**事務局**: 東北大学内（eduroam4s＠rd.cc.tohoku.ac.jp）
**URL**: https://nghsig.jp/eduroam/

対象機関:
- **学校教育法第1条の教育機関**（一条校）のうち小学校以上の学年を扱うもの
- **専修学校**

除外: 附属学校として大学法人等がeduroamを一括運用している機関（ただし大学と分けた独立運用を希望する場合は対象）

**本プロジェクトでの位置づけ**: NGHSIGルートは接続が事業者経由が原則であり、自前RADIUSサーバー運用の方針と合わない可能性がある。eduroam JP本体への参加が第一選択肢であり、NGHSIGルートは最終的な代替手段として位置づける。

## 4. 推奨方針

### 4.1 申請主体と根拠

| 項目 | 内容 |
|------|------|
| **申請主体** | **学校法人コンピュータ総合学園** |
| **参加資格根拠** | 神戸情報大学院大学（学校教育法第83条の大学）を設置する学校法人 |
| **展開対象** | 神戸情報大学院大学（kic.ac.jp）＋ 神戸電子専門学校（kobedenshi.ac.jp） |
| **先行事例** | 学校法人三幸学園（大学＋多数の専門学校を設置する法人が法人名義で参加） |

### 4.2 方針の優先順位

1. **第一選択: eduroam JP本体（NII）に学校法人名義で申請**
   - 根拠: 大学を設置する学校法人としての参加資格
   - 先例: 学校法人三幸学園が同様の構造で参加済み
   - 利点: 自前RADIUSサーバー運用が可能、両校一括管理

2. **代替案1: 神戸情報大学院大学の単独名義で申請し、専門学校を附属校扱いで含める**
   - 根拠: 大学院大学として直接参加資格がある
   - 注意: 附属校扱いの適用可否はNII確認が必要

3. **代替案2: 学校向けeduroam（NGHSIG）経由**
   - 根拠: 専修学校として参加資格あり
   - 制約: 事業者経由接続が原則。自前RADIUS運用の方針との整合性が低い

### 4.3 UPKI電子証明書との整合

UPKI電子証明書発行サービスの申請主体も「学校法人コンピュータ総合学園」を第一候補としており（利用規程 第2条第2号に該当）、eduroam JPの申請主体と統一することで手続きの一貫性が保てる。

詳細: [UPKI-eduroam申請資格調査](../UPKI/UPKI-eduroam申請資格調査.md)

## 5. 申請条件

### 5.1 SINET接続の要否

**SINET加入は不要。** SINETや学認への加入の有無は問われない。

ただし申請手続きが異なる:
| 条件 | 手続き |
|------|--------|
| SINET加入機関 | 機関の長の押印・郵送を省略可 |
| SINET未加入機関 | 機関の長（学長・理事長等）の記名・押印＋郵送が必要 |

**神戸情報大学院大学はSINET6に加入済み**（神戸電子専門学校は未加入）。学校法人コンピュータ総合学園として申請する場合、SINET加入機関として扱われ、機関の長の押印・郵送手続きを省略できる可能性が高い。

### 5.2 費用

**無料。** eduroam JPへの参加に費用はかからず、加入機関への料金徴収もない。

### 5.3 アクセスネットワーク整備

原則として認証サーバーとアクセスネットワーク（Wi-Fi AP等）の整備が必要だが、**加入後1年以内に整備完了する計画があれば例外的に先行申請が可能**。

## 6. 申請手続き

### 6.1 eduroam JP 申請システム

申請は「eduroam JP申請システム」（https://office.eduroam.jp/ ）を通じて行う。

申請の流れ:
1. eduroam JP申請システムにアクセス
2. 加入申請を提出（神戸情報大学院大学はSINET6加入済みのため、押印・郵送省略の可能性あり）
3. NIIによる審査（**2週間〜1ヶ月半**）
4. 承認後、技術担当者を登録し、eduroam JP運用連絡会に参加
5. RADIUS接続設定・テスト

### 6.2 学校法人での一括申請

学校法人として一括申請は可能。ただし**異なるrealmを使用する場合は機関ごとに申請書を提出**し、各機関から技術担当者を選出する必要がある。

本プロジェクトでは以下の2つのrealmを想定:
- `kobedenshi.ac.jp`（神戸電子専門学校）
- `kic.ac.jp`（神戸情報大学院大学）

異なるrealmを使用する場合、1つの参加主体の下に2つのrealmを登録できるか、それとも機関ごとの申請が必要かはNIIへの確認事項。

## 7. 認証方式の選択肢

eduroam参加時に以下の認証方式から選択する:

| 方式 | 概要 | RADIUSサーバー | 備考 |
|------|------|---------------|------|
| 自前RADIUSサーバー構築 | 自機関でRADIUS IdPを構築・運用 | **必要** | 最も柔軟。本プロジェクトの方針 |
| 認証連携IDサービス | 学認（Shibboleth）連携でアカウント発行 | **不要** | 学認加入が前提 |
| ~~代理認証システム (DEAS)~~ | ~~管理者登録のみで運用開始~~ | ~~不要~~ | **2024年3月末で廃止済み** |

### 7.1 認証連携IDサービス（旧DEAS後継）

- URL: https://federated-id.eduroam.jp/
- 旧・代理認証システム（DEAS）と仮名アカウント発行システムの後継
- 学認（Shibboleth）による認証を経てアカウント発行
- RADIUSサーバーの構築・運用が不要
- realmは `(機関ドメイン名).eduroam.jp` 形式

## 8. 学校向けeduroam（NGHSIG）の特徴

eduroam JP本体とは異なる運用ルールがある:

| 項目 | eduroam JP本体 | 学校向けeduroam |
|------|---------------|----------------|
| 運営 | NII | NGHSIG（東北大学内） |
| 対象 | 大学・研究機関 | 一条校＋専修学校 |
| 接続方式 | 直接接続 | **事業者経由が原則** |
| 直接接続 | 可 | 大規模校・自治体のみ |
| 基地局マップ | 個別提出 | 事業者取りまとめ |

## 9. NII への確認事項（優先度順）

### 最優先: 申請主体と展開範囲

1. **申請主体**: 学校法人コンピュータ総合学園として申請し、神戸情報大学院大学と神戸電子専門学校の両方を展開対象にできるか
2. **附属校扱い**: 同一法人の設置校（神戸電子専門学校）を大学の加入申請に含める一括管理が適用できるか
3. **専修学校の扱い**: 学校法人名義であっても神戸電子専門学校（専修学校）へのeduroam提供に制約はあるか

### 技術的確認事項

4. **複数realm**: `kic.ac.jp` と `kobedenshi.ac.jp` の両方を1参加機関の下で登録できるか。それとも realm ごとに別申請が必要か
5. **公開機関名**: 参加機関一覧での掲載名（法人名 or 大学名 or 両方）
6. **site登録**: 1つの参加機関の下で複数site（複数キャンパス）をどう登録するか

### 事前に自機関で確認すべき事項

7. **SINET加入機関としての扱い**: 神戸情報大学院大学はSINET6に加入済みだが、学校法人名義での申請時にSINET加入機関として扱われるかの確認（押印・郵送手続き省略の可否）
8. **機関の長**: 申請書に記名・押印する機関の長は誰か（理事長? 学長?）

### NGHSIG への確認事項（NII経由が不可の場合のみ）

9. **自前RADIUS運用**: 事業者経由ではなく自前RADIUSサーバーで参加可能か
10. **大学法人との関係**: 神戸情報大学院大学と同一法人だが、神戸電子専門学校を独立運用として扱えるか

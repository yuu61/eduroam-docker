# UPKI電子証明書発行サービス・eduroam JP 申請資格調査

> 調査日: 2026-03-16

## 1. 調査目的

学校法人コンピュータ総合学園が、UPKI電子証明書発行サービス利用規程 第2条の対象機関に該当するかを確認し、神戸情報大学院大学と神戸電子専門学校の双方で eduroam を展開する前提で、UPKI と eduroam JP の申請主体をどう整理するべきかを明確にする。

## 2. 結論

- 学校法人コンピュータ総合学園は、UPKI電子証明書発行サービス利用規程 第2条第2号に該当する。
- 該当根拠は、同法人が神戸情報大学院大学を設置している点にある。
- 神戸電子専門学校は専修学校であり、それ単体では第2条第1号または第2号の根拠にならない。
- 神戸情報大学院大学と神戸電子専門学校の両方で eduroam を展開する計画であれば、UPKI の申請主体は `学校法人コンピュータ総合学園` を第一候補とするのが妥当である。
- ただし、eduroam JP 側の参加機関名、公開名、site 登録単位は別途運用判断があり得るため、NII に事前確認を入れるべきである。

## 3. 根拠

### 3.1 法人と設置校

| 項目 | 内容 |
|------|------|
| 法人名 | 学校法人コンピュータ総合学園 |
| 設置校1 | 神戸情報大学院大学 |
| 設置校2 | 神戸電子専門学校 |

- 神戸情報大学院大学の設置者は、学校法人コンピュータ総合学園である。
- 神戸電子専門学校の設置者も、学校法人コンピュータ総合学園である。

出典:

- [神戸情報大学院大学 大学概要](https://www.kic.ac.jp/outline/about/)
- [神戸電子専門学校 学校概要と沿革](https://www.kobedenshi.ac.jp/info/about_us.html)

### 3.2 UPKI利用規程 第2条への当てはめ

UPKI電子証明書発行サービス利用規程では、利用申請可能な機関を次のように定めている。

- 第2条第1号: 大学、短期大学、高等専門学校、大学共同利用機関
- 第2条第2号: 学校法人であって学校教育法第83条に定める大学もしくは同法第115条に定める高等専門学校を設置する機関

該当性は次のとおり。

| 区分 | 該当 | 判断 |
|------|------|------|
| 神戸情報大学院大学 | 該当 | 「大学」として扱われるため、第1号の対象になり得る |
| 神戸電子専門学校 | 非該当 | 専修学校であり、第1号の大学・短大・高専には当たらない |
| 学校法人コンピュータ総合学園 | 該当 | 神戸情報大学院大学を設置する学校法人であり、第2号に当たる |

したがって、今回の申請資格の根拠は `神戸情報大学院大学を設置する学校法人` である点にある。

参照:

- [国立情報学研究所UPKI電子証明書発行サービス利用規程](./国立情報学研究所UPKI電子証明書発行サービス利用規程.md)

### 3.3 神戸電子専門学校単独名義が弱い理由

- 神戸電子専門学校は「専門学校」であり、「高等専門学校」ではない。
- UPKI利用規程 第2条第1号は専修学校を列挙していない。
- 第2条第2号の学校法人要件は、「大学」または「高等専門学校」を設置していることを求めている。
- よって、神戸電子専門学校単独ではなく、大学を設置する同一法人を申請主体に据える方が条文との整合がよい。

## 4. 申請主体の整理

### 4.1 UPKI

UPKI 側は「利用機関」単位で申請し、利用機関ごとに機関責任者、登録担当者、利用管理者を置く建て付けである。

参照:

- [国立情報学研究所UPKI電子証明書発行サービス利用規程](./国立情報学研究所UPKI電子証明書発行サービス利用規程.md)
- [支援システム操作手順書_はじめに](./支援システム操作手順書_はじめに.md)

この前提で比較すると、次の整理になる。

| 観点 | 法人名義 | 神戸電子専門学校単独名義 |
|------|----------|--------------------------|
| 第2条との整合 | 強い | 弱い |
| 神戸情報大学院大学を含む説明 | 容易 | 別立て説明が必要 |
| 複数校展開との整合 | 高い | 低い |
| 体制の一元化 | しやすい | 分散しやすい |

したがって、UPKI の申請主体は `学校法人コンピュータ総合学園` を第一候補とする。

### 4.2 複数ドメインの扱い

UPKI の審査資料では、「利用確認書」は利用機関ごとに発送され、複数ドメインを所有している場合は 1 枚に全ドメイン名が記載されるとされている。

これは、`1利用機関 = 複数ドメイン` の運用を前提にできることを示しており、法人名義で `kic.ac.jp` と `kobedenshi.ac.jp` を扱う整理と相性がよい。

参照:

- [発行前審査方法](./発行前審査方法.md)

### 4.3 eduroam JP

eduroam JP は機関単位での参加を前提にしているため、UPKI を法人名義で申請する場合でも、次の点は NII に確認が必要である。

- 参加機関名を `学校法人コンピュータ総合学園` として扱うのか
- 公開上の参加機関名を `神戸情報大学院大学` または `神戸電子専門学校` として表示できるのか
- 1参加機関の下で複数 site または複数 SSID 提供拠点をどう登録するのか
- `kic.ac.jp` と `kobedenshi.ac.jp` の両ドメインを同一参加主体で扱えるのか

参考:

- [eduroam JP 管理者向け情報](https://www.eduroam.jp/for_admin)
- [eduroam JP 参加機関一覧](https://eduroam.jp/node/375)

## 5. 現時点の推奨方針

### Step 1: UPKI の申請主体を法人名義で整理する

- 申請主体: `学校法人コンピュータ総合学園`
- 申請根拠: 利用規程 第2条第2号
- 機関責任者: 課長職以上または准教授相当以上の常勤教職員
- 登録担当者: 機関責任者から任命
- 利用管理者: 証明書秘密鍵の管理責任者

### Step 2: 対象ドメインと証明書要求を整理する

- eduroam 用 RADIUS サーバーの FQDN を確定する
- `kic.ac.jp` と `kobedenshi.ac.jp` のどちらを利用するか、または両方を使うかを決める
- 現行の技術検証資料は `radius.kobedenshi.ac.jp` を前提としているため、神戸情報大学院大学も対象に含めるなら命名方針を再整理する

参照:

- [eduroam技術検証環境 アーキテクチャ設計書](../architecture.md)

### Step 3: eduroam JP へ事前照会する

NII には、少なくとも次を確認する。

- 申請主体を学校法人コンピュータ総合学園として問題ないか
- 展開対象を神戸情報大学院大学と神戸電子専門学校の両方にできるか
- 参加機関一覧の掲載名をどう扱うか
- 1つの参加主体で複数ドメインを扱えるか

## 6. 確認事項

| 項目 | 現状 | コメント |
|------|------|----------|
| UPKI利用資格 | 確認済み | 学校法人コンピュータ総合学園は第2条第2号に該当 |
| UPKI利用有無 | 要確認 | 既に利用機関として登録済みかは管理画面外から確認できない |
| eduroam JP参加有無 | 未加入（確認済み） | 公開参加機関一覧にも掲載なし |
| SINET接続契約の主体 | 神戸情報大学院大学がSINET6加入済み | 法人名義申請時にSINET加入機関として扱われるかはNIIへ要確認 |
| 対象ドメイン | 要確認 | `kic.ac.jp`、`kobedenshi.ac.jp`、または両方 |
| 公開機関名・site登録単位 | 要確認 | eduroam JP 側の運用判断を確認する必要がある |

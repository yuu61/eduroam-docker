## 1. サーバ証明書管理手順

本章では、サービス参加機関の利用管理者に配付するサーバ証明書の発行・更新・失効及び管理を行う登録担当者の操作方法について記述します。

| 手続きの種別 | 手続きを行う主な機会 |
|---|---|
| **新規証明書発行**（1-1） | 新規にサーバ証明書の発行を必要とする場合。サーバ証明書の記載内容（主体者DN）を変更する場合 |
| **証明書更新発行**（1-2） | サーバ証明書の記載内容（主体者DN以外）を変更する場合。サーバ証明書を継続利用したい場合。失効されたサーバ証明書の再発行を行う場合 |
| **証明書失効**（1-3） | サーバ証明書が不要になった場合や秘密鍵が危殆化した場合 |

### 1-1. サーバ証明書新規発行

以下に記述する手続きにより、サーバ証明書の新規発行を行います。

#### 手順概要

**【利用管理者→登録担当者】**

1. サーバ証明書を申請する際に必要となる鍵ペア及びCSRの作成を行います（1-1-1）
2. サーバ証明書新規発行申請TSVファイルを作成します。TSVファイルに①で生成したCSRを貼り付けます（1-1-2）

**【登録担当者作業】**

3. サーバ証明書新規発行申請TSVファイルを利用管理者から受け取ります。利用管理者からTSVファイルを受け取るときには、利用管理者の本人確認を行ってください（1-1-3）
4. サーバ証明書新規発行申請TSVファイルのアップロードを行ってください（1-1-4）

**【利用管理者作業】**

- サーバ証明書発行受付通知メールを受信します
- サーバ証明書発行受付通知メールの情報を元に、利用管理者用証明書取得URLにアクセスします
- サーバ証明書をダウンロードします

**【登録担当者作業】**

5. ダウンロード完了通知メールを受信（1-1-5）

#### 1-1-1. 利用管理者による鍵ペア・CSRの作成

利用管理者はサーバ証明書の発行申請を行うため事前に鍵ペア及びCSRの作成を行います。

#### 1-1-2. 利用管理者によるサーバ証明書新規発行申請TSVファイル作成

利用管理者は、サーバ証明書新規発行申請TSVファイルを作成します。TSVファイル作成用Webアプリケーション（TSVツール）を提供しておりますので、ご利用ください。作成したTSVファイルを登録担当者宛にメールで送付します。

#### 1-1-3. TSVファイルの受け取り及び審査

登録担当者は、利用管理者からサーバ証明書新規発行申請TSVファイルを受領します。その際、各機関の定められた手続きに従い利用管理者の本人性・実在性を審査してください。

#### 1-1-4. サーバ証明書新規発行申請TSVファイルのアップロード

**手順:**

1. 本システムに接続してください
2. サーバ証明書の[発行・更新・失効]を選択してください
3. 処理内容で[サーバ証明書 発行]を選択してください
4. エラー発生時の動作を選択します:
   - **全件処理を中断する** — 申請データに1件でもエラーがあった場合すべての申請を取りやめる（推奨）
   - **入力値不備のないデータのみ登録する** — エラーがあったデータのみ申請を無視して処理を続行する
5. [参照]を選択してTSVファイルを選択し、[アップロード]を選択してください
6. 審査項目がポップアップされますので、各項目を確認後、問題がなければ[OK]を選択してください

**エラーメッセージと対処方法:**

| エラーメッセージ | 原因 | 対処方法 |
|---|---|---|
| 対象のドメインは許可されていないため発行できません | 審査完了していないため発行不可 | サービス窓口にお問い合わせください |
| 機関の証明書発行有効期限が切れています | 証明書発行承認有効期限を超過 | サービス窓口にお問い合わせください |
| WHOISサーバへの接続に失敗しました | Whoisサーバ接続に失敗 | 時間をおいて再度申請。再度失敗する場合はサービス窓口にお問い合わせください |
| WHOISサーバに登録されている情報と一致しません | Whoisサーバの組織名・登録者名が変更された | サービス窓口にお問い合わせください |
| 主体者DNのLが登録された値と一致しません | 指定されたLの値で申請されていない | 「UPKI証明書 主体者DNにおける ST および L の値一覧」を参照 |
| 主体者DNのSTが登録された値と一致しません | 指定されたSTの値で申請されていない | 「UPKI証明書 主体者DNにおける ST および L の値一覧」を参照 |

7. アップロード終了後、[ログアウト]を選択し、本システムを終了してください

#### 1-1-5. ダウンロード完了通知メール受信

サーバ証明書新規発行依頼者である利用管理者がサーバ証明書のダウンロードを行った場合、本システムより、登録担当者宛にダウンロードが完了したことを通知するメールが自動送信されます。このメールは電子署名されています。

一定期間がたってもダウンロード完了通知メールが受信されない場合は、該当の利用管理者へ電話にて連絡を行い、状況を確認してください。

### 1-2. サーバ証明書更新発行

#### 手順概要

**【利用管理者→登録担当者】**

1. サーバ証明書を更新申請する際に必要となる鍵ペアCSRを作成します（1-2-1）
2. サーバ証明書更新申請TSVファイルを作成します。TSVファイルに①で生成したCSRを貼り付けます（1-2-2）

**【登録担当者作業】**

3. サーバ証明書更新申請TSVファイルを利用管理者から受け取ります。本人性確認を行ってください（1-2-3）
4. サーバ証明書更新申請TSVファイルのアップロード（1-2-4）

**【利用管理者作業】**

- サーバ証明書発行受付通知メールを受信
- 利用管理者用証明書取得URLにアクセス
- サーバ証明書をダウンロード

**【登録担当者作業】**

5. ダウンロード完了通知メールを受信（1-2-5）

**【利用管理者作業】**

- 旧サーバ証明書から新サーバ証明書への置き換え作業
- 登録担当者に新サーバ証明書への置き換え完了通知

**【登録担当者作業】**

6. 利用管理者から新サーバ証明書への置き換え完了通知を受け（1-2-6）
7. ダウンロード完了通知メールに添付されていたファイルをそのまま失効申請TSVファイルとしてアップロード（1-2-7）
8. 旧サーバ証明書失効完了通知メールを受信（1-2-8）

> **注意:** ダウンロード完了通知メール受信から2週間以内に旧サーバ証明書の失効申請が行われないと、本システムより旧サーバ証明書失効依頼のメールが再通知されます。

#### 1-2-1〜1-2-3

新規発行と同様の手順です。更新申請TSVファイルの「失効対象証明書シリアル番号」項目の調べ方は「1-4. サーバ証明書情報取得」を参照してください。

#### 1-2-4. サーバ証明書更新申請TSVファイルのアップロード

「1-1-4」の手順で、[サーバ証明書 更新]を選択してください。

#### 1-2-5. ダウンロード完了通知メール受信

利用管理者が更新したサーバ証明書をダウンロードすると、登録担当者宛にダウンロード完了通知メールが送信されます。旧サーバ証明書が有効の場合、TSVファイル（旧サーバ証明書失効申請TSVファイル）が添付されています。

> 旧サーバ証明書失効申請TSVファイルアップロードを行う日までこのメールは大切に保管してください。

#### 1-2-6. 新サーバ証明書への置き換え完了通知

利用管理者は該当のサーバ証明書の置き換えを行い、登録担当者へサーバ証明書の置き換え完了通知を行います。

#### 1-2-7. 旧サーバ証明書失効TSVファイルのアップロード

ダウンロード完了通知メールに添付のTSVファイル（旧サーバ証明書失効申請TSVファイル）を本システムへアップロードします。

#### 1-2-8. 旧サーバ証明書失効完了通知メール受信

サーバ証明書の失効を行った場合、本システムより、登録担当者と利用管理者宛に失効完了通知メールが送信されます。

#### 1-2-9. 旧証明書失効依頼再通知について

利用管理者による新サーバ証明書のダウンロード完了から2週間経過後、旧サーバ証明書が有効のままであった場合、旧サーバ証明書の失効依頼再通知メールが送信されます。本メールを受信した場合は、利用管理者へ置き換え状況の確認を行い、速やかに失効申請を行ってください。

### 1-3. サーバ証明書失効

#### 手順概要

**【利用管理者作業】**

1. サーバ証明書失効申請TSVファイルを作成します（1-3-1）

**【登録担当者作業】**

2. サーバ証明書失効申請TSVファイルを利用管理者から受け取ります。本人確認を行ってください（1-3-2）
3. サーバ証明書失効申請ファイルのアップロード（1-3-3）
4. サーバ証明書失効完了通知メール受信（1-3-4）

#### 1-3-1〜1-3-2

利用管理者はTSVツールを使用してサーバ証明書失効TSVファイルを作成し、登録担当者宛にメールで送付します。登録担当者は本人性・実在性を審査してください。

#### 1-3-3. サーバ証明書失効申請TSVファイルのアップロード

本システムアップロード時、[サーバ証明書 失効]を選択してください。

#### 1-3-4. サーバ証明書失効完了メール受信

サーバ証明書の失効を行った場合、登録担当者と利用管理者宛に失効完了通知メールが送信されます。このメールは電子署名されています。

### 1-4. サーバ証明書情報取得

#### 主な利用用途

- 過去に申請したサーバ証明書発行・失効・更新履歴や処理状況を知りたい場合
- サーバ証明書の更新・失効にあたり、旧サーバ証明書のシリアル番号を確認したい場合
- 過去に申請したCSR一式、あるいは過去に発行されたサーバ証明書一式を取得したい場合
- 各サーバ証明書の利用管理者情報を知りたい場合

#### 1-4-1. サーバ証明書情報取得手順

「サーバ証明書-証明書情報一括ダウンロード」を選択し、サーバ情報 証明書情報一括ダウンロード画面で[ダウンロード]を選択してください。

## 1. 機関責任者の「在籍確認審査」について

審査時期になりましたら「UPKI電子証明書発行サービス 利用確認書」を発送します（通常は約1年に1回程度です）。

特定記録郵便が届きましたら以下の対応をお願いします:

1. 署名年月日を**西暦**で記載
2. 機関責任者の氏名は「署名のみ」、「署名と押印」または「記名（スタンプ等）と押印」で対応
3. 機関責任者の氏名は、用紙の左上に記載してある氏名と一致させてください
4. 必要事項を記入のうえ、利用確認書の画像データ（スキャナーまたはカメラ撮影し、PDF・JPG等で保存）を以下のメールアドレスに、認証局の郵便物発送から**30日以内**にお送りください

**送付先:** upki [at] secom.co.jp

> **注意:**
> - 利用確認書に記載のランダム値は「機関責任者の在籍確認」についてのみに使用するものであり、ドメイン審査で使用するランダム値とは異なります
> - 機関責任者の変更を予定している場合または手続き中の場合は、新機関責任者の役職と氏名をご連絡ください
> - 変更対応後、新たに利用確認書を送付します
> - 「UPKI電子証明書発行サービス 利用確認書」は利用機関ごとに発送しますので複数ドメインを所有している場合は1枚に全ドメイン名が記載されます

## 2. 「ドメイン審査」（ドメイン使用許諾承認）について

ドメイン審査の方法について、下記よりご選択ください。

### 2-1. メール認証

以下の5つのメールアドレスのいずれかにランダム値を送信し、認証局に返信いただく方法です（**2週間以内**のご返信をお願いします）。

- `admin@<ドメイン名>`
- `administrator@<ドメイン名>`
- `webmaster@<ドメイン名>`
- `hostmaster@<ドメイン名>`
- `postmaster@<ドメイン名>`

> 上記5つのメールアドレスが無い場合は作成をお願いします。
>
> **例:** 申請ドメインが `zzz.ac.jp` の場合
> - OK: `admin@zzz.ac.jp`, `administrator@zzz.ac.jp` 等
> - 不可: `admin@ml.zzz.ac.jp`（サブドメインを含んでいるものは不可）
>
> 認証局へ返信する際に使用するメールアドレスは上記以外のメールアドレス（個人メールアドレスやグループメール）から送られても問題ありません。

### 2-2. DNSメール認証（メールアドレスをDNSへ登録）

DNSレコードにメールアドレスを登録し、そこにランダム値を送信して認証局に返信いただく方法です（**2週間以内**のご返信をお願いします）。

認証局からのメールを受信できるメールアドレスであれば、どのようなメールアドレスでも問題ありません。

**TXTレコード例:**

```
_validation-contactemail.zzz.ac.jp. 3600 IN TXT "yamamoto@zzz.ac.jp"
```

> 審査完了後、削除しても残していても問題ありません。残しておけば次回の審査で利用できます。

### 2-3. DNS認証（認証局指定のランダム値をDNSへ登録）

DNSレコードに「認証局指定のランダム値」を設定していただく方法（**2週間以内**のご対応をお願いします）。

**TXTレコード例:**

```
zzz.ac.jp. 3600 IN TXT "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

> - ランダム値は認証局から送付したものを「そのまま」登録してください
> - 登録後、削除することを推奨します（次回審査時に前回のレコードが残っていると分かりにくくなる恐れがあります）
> - `xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` の部分は16進数（0〜9およびA〜Fが含まれます）のランダム値です

## 3. WHOISを使用したドメイン審査廃止（2025年1月15日以降）対応について

2024年10月25日より、WHOIS以外の方式を利用した臨時のドメイン審査を行っています。

従来WHOISを使用したドメイン審査を行った利用機関は、今後の審査スケジュールがずれる場合があります。

**例1: 2024年5月31日以前に前回の審査が完了している場合**

| 日付 | イベント |
|------|------|
| 2024-05-01 | ドメイン審査および在籍確認審査完了 → 本来「2025-06-02」まで発行可だが「2025-01-13」まで発行可に変更 |
| 2024-11-20 | 臨時のドメイン審査および在籍確認審査完了 → 「2025-12-21」まで発行可 |
| 2025-10-01 | 次回審査の案内開始 |

**例2: 2024年6月1日以降に前回の審査が完了している場合**

| 日付 | イベント |
|------|------|
| 2024-06-01 | ドメイン審査および在籍確認審査完了 → 本来「2025-07-02」まで発行可だが「2025-01-13」まで発行可に変更 |
| 2024-11-20 | 臨時のドメイン審査のみ完了 → 「2025-12-21」まで発行可 |
| 2025-05-01 | 次回審査の案内開始（臨時でドメイン審査のみのため、10か月後に案内） |

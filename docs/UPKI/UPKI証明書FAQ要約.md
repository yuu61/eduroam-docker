# UPKI電子証明書発行サービス FAQ要約

出典: https://certs.nii.ac.jp/faq/q1 ~ q7

本ドキュメントはeduroam導入に関連性の高い情報を中心に整理したもの。

---

## 1. サービス概要 (q1)

| 項目 | 内容 |
|------|------|
| 費用 | 有償事業（料金は利用細則別紙に記載） |
| 構成員数の算定 | **常勤の教員と研究者の合計**（常勤職員・学生は含まない） |
| 学校法人の場合 | 法人として一括申請。構成員数は法人＋傘下学校の常勤教員・研究者合計 |
| 利用期間 | 年度ごとに更新。2〜3月頃に継続意思確認あり |
| 継続意思確認を怠った場合 | **全証明書が失効** |
| 請求書 | サービス開始後に経理担当者へ発送（複数ドメインは一括請求） |
| 民間企業の代理申請 | 不可 |
| フィードバック | 年度ごとにアンケート（発行・失効枚数、本人確認方法等を報告） |

## 2. 利用申請 (q2)

### 提出書類
- **サービス利用申請書**
- **ドメイン申請書**
- UPKI申請システムから申請

### 申請に関する注意点

| 項目 | 内容 |
|------|------|
| 利用開始までの所要日数 | **最短40日**。前々月20日までに押印済み書類がUPKI担当に必要 |
| ドメイン数の制限 | なし（追加ドメイン1つごとに利用料が発生） |
| 登録担当者 | 複数設定可能（多くとも10名程度が推奨） |
| サブドメインごとの権限分割 | **不可** |
| メールアドレスのドメイン | 原則として登録ドメインを使用（例外は事前相談） |

### 本人性確認・実在性確認

**実在性確認**（組織在籍確認）:
- 職員名簿記載、職員証提示等

**本人性確認**（申請者本人による申請行為の確認）:
- 対面確認、電話確認、ID/パスワードシステム利用等

確認実施手順調査票はUPKI申請システムで作成時に審査される。

### 変更手続き
- 申請内容・機関名・機関責任者・経理担当者の変更 → **変更申請書**
- 登録担当者の追加削除等 → **変更届**
- 機関名変更時、既発行証明書は自動失効されない（必要に応じて更新申請＋失効申請）

## 3. 証明書仕様 (q3)

| 項目 | 内容 |
|------|------|
| 発行枚数の制限 | **なし** |
| 鍵長（RSA / sha256WithRSAEncryption） | 2048bit |
| 鍵長（ECDSA / ecdsa-with-SHA384） | 384bit |
| 有効期間の指定 | **不可**（開始日・終了日とも指定できない） |
| クライアント証明書 | 個人以外に役職・部署等にも発行可能 |

## 4. CSR作成 (q4)

| 項目 | 内容 |
|------|------|
| CSRルール | 「CSRに関するルール」ドキュメント参照 |
| 一括申請 | TSVファイルで最大99件（処理時間は約30分） |
| IPアドレスでの発行 | **不可**（ドメイン申請書記載のドメインが対象） |
| 仮想サーバ（VMware等） | 仮想環境ごとに申請が必要 |
| バーチャルホスト | 1枚で対応可能（dNSNameに最大8件のホスト名を列挙） |
| 同一FQDNの複数サーバ | ①1枚を複数機器にコピー ②サーバごとにCSR作成 ③dNSNameに共通FQDN指定 |
| macOS | LibreSSLではなく別途OpenSSLが必要（Homebrew推奨） |

## 5. サーバ証明書の利用環境 (q5)

### eduroam RADIUSサーバでの利用

> RADIUSサーバやPOP3/SMTPサーバでも利用可能だが、**サポート対象外**。
> SSL/TLSサーバ認証での利用が前提。

| 項目 | 内容 |
|------|------|
| 有効期間 | **396日間**（2020年8月25日以降発行分） |
| グローバルIPアドレス | 不要（ただしOCSP/CRL検証にはインターネット接続が必要） |
| サーバリプレース時 | FQDNが同じなら継続利用可能（鍵ペア紛失時は更新申請） |
| 古い端末のSHA-2対応 | SHA-1→SHA-2クロスルート証明書を提供 |

## 6. トラブルシューティング (q6)

| 症状 | 原因・対処 |
|------|-----------|
| 発行通知メールが届かない | 通常数分で送信。十数分待っても届かなければ認証局内エラーの可能性 |
| 「有効な証明書情報がありません」 | アクセスPINの入力ミス（0/O、1/lの見間違えに注意） |
| 「無効なフラグが指定されました」 | CSP選択ミス。「Microsoft Enhanced Cryptographic Provider v1.0」に変更 |
| サーバ証明書ダウンロード失敗 | 取得URLの有効期限は**30日**。期限切れ時は登録担当者に取得代行を依頼 |
| 「セキュリティ証明書には問題があります」 | 中間CA証明書のインストール漏れ |
| 「証明書は失効しています」 | CRLによる失効確認。ブラウザのCRL設定を確認 |
| HTTPS設定サイトでコンテンツ未表示 | すべての要素をHTTPSで指定する必要あり（Mixed Content問題） |
| UPKI証明書使用時のアクセス遅延 | セコムOCSPレスポンダ障害の可能性。お知らせページで障害情報を確認 |

## 7. UPKI申請システム (q7)

- 利用期間更新申請と機関責任者変更は同時申請可能
  - 機関情報変更には原則として**変更前の機関責任者の署名・印**が必要
  - 新任者の場合は組織印が望ましい（私印なら電話確認あり）
  - 更新申請を同時提出する場合は**新任の機関責任者の署名・印**
- 利用期間更新と経理担当者変更も同時申請可能
  - 請求書は例年6月頃の最新登録情報に基づいて郵送

## eduroam導入プロジェクトへの影響まとめ

1. **UPKI証明書はRADIUSサーバで利用可能**（ただしサポート対象外）
2. **申請から利用開始まで最短40日**かかるため、早期申請が必要
3. 有効期間は**396日間**で指定不可。年次更新を運用計画に含める
4. CSR作成時は**2048bit RSA**で作成し、FQDNベースで申請（IPアドレス不可）
5. 年度ごとの**継続意思確認**を怠ると全証明書が失効するため、運用カレンダーに組み込む
6. 構成員数は**常勤教員・研究者の合計**で算定（学生は含まない）

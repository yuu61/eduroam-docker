# eduroam JP 申請資格・参加条件 調査結果

調査日: 2026-03-16

## 1. eduroam JP の運営体制

eduroam JPは**国立情報学研究所（NII）**が運営する、教育・研究機関のキャンパス無線LANの相互利用を実現する国際的なネットワークローミングサービス。世界100以上の地域が参加し、国内では**478機関**が参加している（2026年3月時点）。

## 2. 参加対象機関

### 2.1 eduroam JP 本体（NII運用）

対象: **大学等高等教育機関や研究機関**

参加機関の内訳:
| カテゴリ | 機関数（概算） |
|---------|--------------|
| 国立大学 | 約50 |
| 公立大学 | 約40 |
| 私立大学 | 約180以上 |
| 高等専門学校 | 約50 |
| 研究機関・その他 | 約30 |

**専門学校（専修学校専門課程）の参加実績は確認できない。**

### 2.2 学校向けeduroam（NGHSIG運用）

初等・中等教育機関や専修学校を対象とした別ルートが存在する。

**運営**: セキュア公衆無線LANローミング研究会（NGHSIG）
**事務局**: 東北大学内（eduroam4s＠rd.cc.tohoku.ac.jp）
**URL**: https://nghsig.jp/eduroam/

対象機関:
- **学校教育法第1条の教育機関**（一条校）のうち小学校以上の学年を扱うもの
- **専修学校**

除外: 附属学校として大学法人等がeduroamを一括運用している機関（ただし大学と分けた独立運用を希望する場合は対象）

### 2.3 神戸電子専門学校の場合

神戸電子専門学校は専修学校であるため、以下の2つのルートが考えられる:

| ルート | 根拠 | 備考 |
|--------|------|------|
| **eduroam JP本体（NII）** | 設置者が学校法人コンピュータ総合学園であり、神戸情報大学院大学（大学）も設置 | 大学を設置する学校法人として一括申請が可能か要確認 |
| **学校向けeduroam（NGHSIG）** | 専修学校は明確に対象 | 接続は事業者経由が原則 |

**推奨**: まずeduroam JP本体（NII）への申請を検討する。architecture.mdに記載の通り、申請主体を「学校法人コンピュータ総合学園」とすれば、大学を設置する学校法人としてeduroam JP本体に参加できる可能性が高い。NGHSIGルートは事業者経由接続が原則であり、自前RADIUS運用の方針とは合わない可能性がある。

## 3. 申請条件

### 3.1 SINET接続の要否

**SINET加入は不要。** SINETや学認への加入の有無は問われない。

ただし申請手続きが異なる:
| 条件 | 手続き |
|------|--------|
| SINET加入機関 | 機関の長の押印・郵送を省略可 |
| SINET未加入機関 | 機関の長（学長・理事長等）の記名・押印＋郵送が必要 |

### 3.2 費用

**無料。** eduroam JPへの参加に費用はかからず、加入機関への料金徴収もない。

### 3.3 アクセスネットワーク整備

原則として認証サーバーとアクセスネットワーク（Wi-Fi AP等）の整備が必要だが、**加入後1年以内に整備完了する計画があれば例外的に先行申請が可能**。

## 4. 申請手続き

### 4.1 eduroam JP 申請システム

申請は「eduroam JP申請システム」（https://office.eduroam.jp/ ）を通じて行う。

申請の流れ:
1. eduroam JP申請システムにアクセス
2. 加入申請を提出（SINET未加入の場合は機関の長の記名・押印＋郵送）
3. NIIによる審査（**2週間〜1ヶ月半**）
4. 承認後、技術担当者2名を登録
5. RADIUS接続設定・テスト

### 4.2 学校法人での一括申請

学校法人として一括申請は可能。ただし**異なるrealmを使用する場合は機関ごとに申請書を提出**し、各機関から技術担当者2名を選出する必要がある。

## 5. 認証方式の選択肢

eduroam参加時に以下の認証方式から選択する:

| 方式 | 概要 | RADIUSサーバー | 備考 |
|------|------|---------------|------|
| 自前RADIUSサーバー構築 | 自機関でRADIUS IdPを構築・運用 | **必要** | 最も柔軟。本プロジェクトの方針 |
| 認証連携IDサービス | 学認（Shibboleth）連携でアカウント発行 | **不要** | 学認加入が前提 |
| ~~代理認証システム (DEAS)~~ | ~~管理者登録のみで運用開始~~ | ~~不要~~ | **2024年3月末で廃止済み** |

### 5.1 認証連携IDサービス（旧DEAS後継）

- URL: https://federated-id.eduroam.jp/
- 旧・代理認証システム（DEAS）と仮名アカウント発行システムの後継
- 学認（Shibboleth）による認証を経てアカウント発行
- RADIUSサーバーの構築・運用が不要
- realmは `(機関ドメイン名).eduroam.jp` 形式

## 6. 学校向けeduroam（NGHSIG）の特徴

eduroam JP本体とは異なる運用ルールがある:

| 項目 | eduroam JP本体 | 学校向けeduroam |
|------|---------------|----------------|
| 運営 | NII | NGHSIG（東北大学内） |
| 対象 | 大学・研究機関 | 一条校＋専修学校 |
| 接続方式 | 直接接続 | **事業者経由が原則** |
| 直接接続 | 可 | 大規模校・自治体のみ |
| 基地局マップ | 個別提出 | 事業者取りまとめ |

## 7. 確認が必要な事項

### NII（eduroam JP本体）への確認事項

1. **申請主体**: 学校法人コンピュータ総合学園として申請し、神戸電子専門学校（専修学校）もサービス対象にできるか
2. **複数ドメイン**: kic.ac.jp と kobedenshi.ac.jp の両方を1参加機関の下で登録できるか
3. **専修学校の扱い**: eduroam JP本体で専修学校が対象外の場合、NGHSIGへの案内となるか

### NGHSIG への確認事項（NII経由が不可の場合）

1. **自前RADIUS運用**: 事業者経由ではなく自前RADIUSサーバーで参加可能か
2. **大学法人との関係**: 神戸情報大学院大学と同一法人だが、独立運用として扱えるか
3. **申請手続き**: 具体的な参加申請の流れと必要書類
